## 1. 页面概述

消息中心是游寄后台管理系统的统一消息聚合页面，汇集所有业务通知与系统公告，支持按消息类型筛选、未读/已读状态管理、一键已读等操作。运营人员通过消息中心实时掌握订单分配、柜机异常、退款审核等关键业务动态。

**访问入口**：
- 顶部导航栏「消息中心」图标（铃铛，带未读角标）
- 侧边栏菜单无直接入口（消息中心属于顶栏全局入口，非侧边栏菜单项）

**访问权限**：所有登录用户均可访问，消息内容按用户角色和权限过滤。

## 3. 顶部 Tab 栏

### 3.1 Tab 结构

| Tab | 说明 | 默认 |
|-----|------|------|
| 消息通知 | 业务消息通知，含异常监控和消息提醒两类；支持分类筛选 | **默认选中** |
| 系统公告 | 系统版本更新、功能上线等公告；无分类筛选 | — |

### 3.2 Tab 切换行为

| 操作 | 行为 |
|------|------|
| 点击「消息通知」Tab | 切换到消息通知列表，展示筛选区、只看未读、一键已读等控件；重置筛选条件为默认值 |
| 点击「系统公告」Tab | 切换到系统公告列表，隐藏筛选区、只看未读、一键已读等控件；仅展示公告列表 |
| 切换 Tab 时 | 未读角标自动更新；筛选条件、分页均重置为默认 |

### 3.3 未读角标

- 每个 Tab 标签右侧展示未读数量角标（红色小圆标）
- 消息通知角标 = 所有消息通知中 `isRead = false` 的数量
- 系统公告角标 = 所有系统公告中 `isRead = false` 的数量
- 未读数为 0 时角标隐藏

### 3.4 Tab 右侧操作区

| 操作 | 说明 |
|------|------|
| 「绑定公众号，随时掌握柜机动态 >」 | 文字链接，点击弹出微信绑定弹窗（见第 9 节） |
| 「消息设置」 | 文字链接，点击跳转至「消息设置」页面 |

## 4. 筛选区（仅消息通知 Tab）

### 4.1 筛选分类

筛选区分为两个筛选维度，各自独立，组合使用（AND 关系）：

**异常监控：**

| Chip | 对应 sub 类型 | 说明 |
|------|-------------|------|
| 全部 | — | 不筛选，展示所有异常监控类消息 |
| 柜机故障告警 | `locker_fault` | 柜机传感器通信异常等硬件故障通知 |
| 行李超时未领取 | `storage_overtime` | 寄存到期后超时未领取的升级告警 |
| 订单超时未完成 | `order_overtime` | 订单创建后超时未完成的提醒 |
| 柜机即将满载 | `locker_full` | 柜机使用率接近满载的预警 |
| 柜机授权即将到期 | `auth_expire` | 柜机授权即将到期的续期提醒 |

**消息提醒：**

| Chip | 对应 sub 类型 | 说明 |
|------|-------------|------|
| 全部 | — | 不筛选，展示所有消息提醒类消息 |
| 订单分配 | `order_assign` | 订单分配通知 |
| 订单转移 | `order_transfer` | 订单从其他成员转移过来的通知 |
| 行李寄存到期提醒 | `storage_expire` | 客户行李寄存即将到期的提醒 |
| 退款申请待审核 | `refund_review` | 用户申请退款、待审核的通知 |
| 异常件处理完成 | `ticket_done` | 异常工单处理完毕的通知 |

### 4.2 Chip 交互规则

| 操作 | 行为 |
|------|------|
| 初始状态 | 异常监控「全部」选中 + 消息提醒「全部」选中 |
| 点击 Chip | 该维度切换为当前 Chip 选中（单选），列表立即刷新 |
| 已选中的 Chip 再次点击 | 保持选中（不取消），不切换回「全部」 |
| 两个维度同时筛选 | AND 关系，只展示同时满足两个维度条件的消息 |
| Chip 上的红色角标 | 展示该分类下未读消息数量；当该 Chip 选中时，角标变为蓝色 |

### 4.3 筛选逻辑

- 异常监控筛选 + 消息提醒筛选为 **AND 关系**
- 任一维度选中「全部」时，该维度不限制
- 筛选后分页重置为第 1 页

## 5. 列表头区域

### 5.1 消息列表标题

| 元素 | 说明 |
|------|------|
| 列表标题 | 消息通知 Tab 下显示「消息列表」，系统公告 Tab 下显示「系统公告」 |
| 消息计数 | 格式「共计 N 条消息」；N = 当前筛选条件下的消息总数 |

### 5.2 操作控件（仅消息通知 Tab）

| 控件 | 类型 | 说明 |
|------|------|------|

系统公告 Tab 下仅展示列表标题和消息计数，不展示「只看未读消息」和「一键已读」控件。

## 6. 消息列表（消息通知 Tab）

### 6.1 列表项结构

每条消息由以下元素构成：

```
┌─────────────────────────────────────────────────────────────────┐
│ ●  【订单分配】张伟、李娜、王强等5个订单分配给你...  14:00     │
│ ○  【订单转移】订单 YJ20260702020 从王强转移给你...  16:00     │
└─────────────────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 未读圆点 ● | 蓝色实心圆（8×8px），表示未读；已读消息圆点透明隐藏 |
| 消息标题 | 展示消息完整标题；已读消息标题颜色变灰（`ink-muted`）、字重变常规（`font-normal`） |
| 时间 | 格式 `YYYY-MM-DD HH:mm`，使用 mono 字体，右对齐 |

### 6.2 列表项交互

| 操作 | 行为 |
|------|------|
| 点击列表项 | 将该消息标记为已读；圆点消失，标题变灰 |
| hover 列表项 | 行背景变为浅灰色（`#FAFBFC`），提供视觉反馈 |
| 点击后 | 不弹出详情弹窗，仅标记已读（消息内容已在标题中完整展示） |

### 6.3 消息类型与 mock data 示例

| 消息标题（示例） | 分类 | sub 类型 |
|-----------------|------|---------|

## 10. 标记已读交互

### 10.1 单条标记已读

| 触发 | 行为 |
|------|------|
| 点击消息列表项（消息通知） | 该条消息 `isRead` 变为 `true`；列表即时刷新，圆点消失、标题变灰；顶栏未读角标 -1 |
| 点击公告列表项（系统公告） | 该条公告 `isRead` 变为 `true`；列表即时刷新，圆点消失、标题变灰；顶栏未读角标 -1 |

### 10.2 一键已读

| 触发 | 行为 |
|------|------|
| 点击「一键已读(N)」按钮 | 将**当前筛选结果**中所有未读消息标记为已读；Toast 提示「已全部标记为已读」；列表即时刷新；顶栏未读角标更新 |

> **注意**：一键已读只影响当前筛选条件下的消息，不影响被筛选排除的未读消息。

## 13. 数据说明

| 数据项 | 规则 |
|-------|------|
| 消息 ID | 唯一标识，用于标记已读操作 |
| 消息分类 | `category`：`exception`（异常监控）/ `reminder`（消息提醒） |
| 消息子类型 | `sub`：如 `locker_fault`、`order_assign` 等，用于 Chip 筛选 |
| 未读状态 | `isRead`：`true` 已读 / `false` 未读 |
| 消息时间 | 格式 `YYYY-MM-DD HH:mm`，当天消息可省略日期仅显示时间 |
| 消息标题 | 完整展示业务信息，包含关键字段（订单号、柜机编号、客户手机号脱敏等） |
| 系统公告内容 | 多条更新项以换行符 `\n` 分隔，支持序号列表格式 |

## 14. 异常场景处理

| 场景 | 表现 | 处理方案 |
|------|------|---------|
| 消息列表为空（当前筛选无匹配） | 列表区展示空状态图标（inbox 图标）+ 「暂无消息」文字 | 引导用户切换筛选条件或查看其他 Tab |
| 筛选后当前页无数据 | 页码自动退回到有效页（如原在第 3 页、筛选后仅剩 2 页，则显示第 2 页） | 前端自动处理 |
| 系统公告无数据 | 同上，展示「暂无消息」空状态 | — |
| 一键已读时无未读消息 | 按钮隐藏（N=0 时不可见） | 前端根据未读计数控制显隐 |

## 15. 交互规则汇总

| 规则 | 说明 |
|------|------|
| Tab 切换重置筛选 | 切换消息通知/系统公告 Tab 时，筛选条件恢复默认，分页重置 |
| 筛选 Chips 单选 | 异常监控和消息提醒两个维度各自单选，维度间 AND 关系 |
| 筛选后分页重置 | 变更任意筛选条件时自动跳回第 1 页 |
| 标记已读即时生效 | 点击列表项或一键已读后，列表即时刷新，无需二次确认 |
| 一键已读作用范围 | 仅影响当前筛选结果中的未读消息，不影响被筛选排除的消息 |
| 系统公告无复杂交互 | 无筛选、无分页、无一键已读；仅支持点击标记已读 |
| 「只看未读」与 Chip 筛选联动 | 「只看未读」勾选在现有 Chip 筛选结果上进一步过滤 |
| Chip 未读角标 | 始终展示该分类下的实时未读数（不受当前选中的其他 Chip 影响） |
| 顶栏角标全局同步 | 页面内任何已读操作均同步更新顶栏铃铛角标 |
| 手机号脱敏 | 消息标题中手机号展示为 `138****5678` 格式 |
