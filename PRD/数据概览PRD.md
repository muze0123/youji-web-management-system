## 一、页面概述

### 1.1 页面目标

数据概览是运营查看核心经营数据的页面，一期聚焦**基础经营指标 + 趋势 + 结构拆分**，帮助运营了解业务规模、健康度与入口迁移进展，支撑日常运营与对上汇报。一期只做基础指标与简单图表，不做复杂 BI（数据大盘为 V2）。

### 1.2 页面在系统中的位置

- 属后台左侧导航"数据概览"（原总纲"数据统计"）一级菜单；
- 面向管理员与运营员（均可查看）；
- 数据来源：订单 Order（结合柜点 Cabinet、格口 Locker），口径与订单看板一致。

### 1.3 设计与布局

- 遵循 `design.md`：后台三段布局（顶部栏 + 侧边导航 + 主内容区），当前导航"数据概览"高亮；
- 主内容区结构：面包屑 + 页面标题 + 时间范围筛选 + 核心指标卡 + 趋势图表 + 结构拆分；
- 视觉、组件、语义色一律引用 `design.md`；数字用 mono 字体。

## 三、区块详细规格

### 3.1 区块 A：页面头

| 组件 | 内容 |
|------|------|
| 面包屑 | 数据概览 |
| 页面标题 | "数据概览" |
| （可选）说明 | "经营数据统计，数据每隔几分钟更新" + 最近更新时间 |

### 3.2 区块 B：时间范围筛选

| 组件 | 选项/字段 | 说明 |
|------|-----------|------|
| 快捷时间 | 今日 / 昨日 / 近 7 天 / 近 30 天 / 本月 | 默认"近 7 天" |
| 自定义日期范围 | 起止日期选择 | 与快捷时间互斥 |
| 业务类型（可选） | 全部 / 行李寄存 / 行李寄送 | 「常见逻辑补充」按业务筛选指标 |
| 柜机（可选） | 全部 / 指定柜机 | 「常见逻辑补充」按柜机筛选 |

**交互**：

| 操作 | 响应 |
|------|------|
| 选快捷时间 | 全页指标/图表按该时间刷新 |
| 选自定义范围 | 按范围刷新 |
| 切业务/柜机维度 | 联动刷新指标与图表 |

### 3.3 区块 C：核心指标卡（6 项）

以指标卡横向排列（响应式换行），每卡：指标名 + 主数值（mono 大字）+ 环比（可选）+ 小图标。

| 指标 | 定义 | 展示 |
|------|------|------|
| 订单量 | 所选周期订单总数 | 数值 + 环比 |
| 营业收入 | 所选周期实收金额（元） | 金额（mono）+ 环比 |
| 柜点使用率 | 已占用格口 / 可用格口 | 百分比 |
| 异常率 | 异常订单 / 总订单 | 百分比，异常偏高时 danger 提示 |
| 寄存/寄送结构 | 两类业务订单占比 | 寄存 X% · 寄送 Y% |
| 自有承接占比 | 自有小程序订单 / 总订单 | 百分比（重点，见 3.6） |

**环比说明**：「常见逻辑补充」环比为与上一周期对比，上升/下降用 success/danger 箭头，仅作趋势参考。

**交互**：

| 操作 | 响应 |
|------|------|
| 点击指标卡 | 「常见逻辑补充」下钻到对应明细（如点订单量 → 跳订单看板并带时间筛选） |
| 异常率偏高 | 数值用 danger 色 + 提示图标 |

### 3.4 区块 D：趋势图表

| 图表 | 说明 |
|------|------|
| 单量趋势 | 折线/柱状，按日展示所选周期订单量；可叠加寄存/寄送分线 |
| 收入趋势 | 折线/柱状，按日展示所选周期实收 |

**交互**：

| 操作 | 响应 |
|------|------|
| hover 数据点 | 展示该日具体数值（tooltip） |
| 切换寄存/寄送分线（可选） | 图表按业务拆分展示 |
| 点击某日数据点 | 「常见逻辑补充」下钻该日明细 |

> 原型实现：本项目原型禁用未列出的第三方库（见 claude.md），图表用轻量 SVG/Canvas 或 mock 静态图形呈现（见待确认）。

### 3.5 区块 E：结构拆分

按维度展示占比（饼图/环形/条形 + 数据列表）：

| 维度 | 说明 |
|------|------|
| 业务类型 | 行李寄存 vs 行李寄送 |
| 订单来源 | 自有小程序 vs 小铁小程序 |
| 柜型 | 小/中/大格口订单占比 |
| 柜机 Top | 订单量 Top N 柜机排行 |

**交互**：hover 展示具体占比；点击某项下钻明细（可选）。

### 3.6 区块 F：来源占比强调区（重点指标）

**单独强调"自有小程序承接占比"**，因为它支撑"用户入口从小铁迁移到自有"这一战略决策（呼应 BRD/产品规划）。

| 组件 | 内容 |
|------|------|
| 自有 vs 小铁占比 | 大号百分比 + 对比条/环形 |
| 趋势 | 自有承接占比随时间变化趋势（是否在上升） |
| 说明 | "自有小程序承接占比反映入口迁移进展" |

## 五、业务规则

- 一期为基础统计，不做复杂 BI（数据大盘 V2）；
- 数据来自订单，口径与订单看板一致（本地映射状态、金额分转元、来源区分自有/小铁）；
- **来源占比是重点指标**——支撑入口迁移决策；
- 柜点使用率依赖格口占用数据（来自小铁同步，存在延迟，标注时效）；
- 权限：管理员与运营员均可查看数据概览。
