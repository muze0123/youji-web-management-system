## 一、模块概述

### 1.1 模块目标

集中管理异常与售后工单：从订单标记异常、接收用户端售后申请、流转处理、留痕记录，确保三条 P0 异常底线（支付成功锁柜失败、取件超时补费、开柜异常）有跟踪、有闭环、有证据。

### 1.2 工单来源

| 来源 | 说明 |
|------|------|
| 运营标记 | 运营在订单管理"标记异常"生成 |
| 用户售后 | 用户端"客服/售后"提交，自动生成工单 |
| 系统生成 | 「常见逻辑补充」系统检测到锁柜失败等异常自动生成 |

### 1.3 页面构成

| 页面 | 说明 |
|------|------|
| 工单列表 | 筛选 + 工单表格 + 分页 |
| 工单详情/处理面板 | 抽屉/弹窗，查看与处理 |
| 标记异常弹窗 | 从订单管理触发，创建工单 |

### 1.4 入口

- 侧边导航"异常工单"；
- 工作台待办（待处理工单、锁柜失败、售后待处理）带筛选跳入；
- 订单管理"标记异常"生成工单；订单详情"查看关联工单"跳入。

### 1.5 设计与布局

遵循 `design.md`：后台三段布局，侧边导航"异常工单"高亮；主内容区 = 面包屑 + 标题 + 筛选区 + 工单表格 + 分页。视觉、组件、语义色引用 design.md；工单号/订单号/时间用 mono。

## 二、工单状态与异常类型

### 2.1 工单状态

| 状态 | 含义 | Badge 色 |
|------|------|----------|
| 待处理 | 新建未处理 | warning |
| 处理中 | 已受理处理中 | info |
| 已处理 | 处理完成 | success |
| 已关闭 | 关闭归档 | ink-muted 灰 |

流转：待处理 → 处理中 → 已处理 → 已关闭。

### 2.2 异常类型

| 类型 | 说明 |
|------|------|
| 延误 | 寄送延误 |
| 损坏 | 物品损坏 |
| 遗失 | 物品遗失 |
| 取件码/存储码错误 | 凭证问题 |
| 支付异常 | 支付相关 |
| 锁柜失败 | 支付成功但锁柜失败（P0） |
| 开柜异常 | 开柜失败/码失效（P0） |
| 其他 | 其他异常 |

## 三、页面一：工单列表

### 3.1 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"异常工单" |
| B 筛选区 | 工单状态、异常类型、来源、日期、关键词搜索 + 查询/重置 |
| C 工单表格 | 主体，含操作列 |
| D 分页 | 右下角 |

### 3.2 区块 B：筛选

| 筛选项 | 选项 |
|--------|------|
| 工单状态 | 全部/待处理/处理中/已处理/已关闭 |
| 异常类型 | 全部 + 第 2.2 各类型 |
| 来源 | 全部/运营标记/用户售后/系统生成 |
| 日期范围 | 创建时间，默认近 7 天 |
| 关键词 | 工单号 / 关联订单号 |
| 操作 | 查询/重置 |

**交互**：设置筛选查询刷新；重置清空；工作台带筛选进入自动应用（如状态=待处理）。

### 3.3 区块 C：工单表格

| 列 | 说明 | 对齐 |
|----|------|------|
| 工单号 | 唯一标识（mono） | 左 |
| 关联订单 | 订单号，可跳订单详情（mono） | 左 |
| 异常类型 | 枚举 | 左 |
| 来源 | 运营标记/用户售后/系统 | 左 |
| 工单状态 | Badge 用色 | 左 |
| 处理人 | 当前处理人 | 左 |
| 创建时间 | mono | 左 |
| 更新时间 | mono | 左 |
| 操作 | 处理、查看 | 左 |

**交互**：

| 操作 | 响应 |
|------|------|
| 点击"处理"/"查看"/行 | 打开工单详情/处理面板 |
| 点击关联订单 | 跳订单详情 |
| 排序 | 按创建/更新时间 |
| hover 行 | bg-hover 高亮 |

### 3.4 页面状态

加载中（骨架屏）、正常、空结果、网络异常。P0 类型（锁柜失败/开柜异常）待处理工单「常见逻辑补充」可 danger 强调。

## 四、页面二：工单详情/处理面板

### 4.1 目标

查看工单完整信息、处理流转、留痕记录。抽屉（Drawer）或弹窗呈现。

### 4.2 内容分组

| 分组 | 内容 |
|------|------|
| 工单信息 | 工单号、关联订单、异常类型、来源、状态、创建/更新时间 |
| 关联订单摘要 | 订单号、业务类型、柜机、订单状态（点击进订单详情） |
| 用户诉求 | 用户售后申请的问题描述、图片（来自用户端售后时展示） |
| 处理记录 | 处理备注留痕列表，每条含操作人 + 时间 + 内容，可追加 |
| 处理操作 | 变更状态、添加备注、指派处理人、（可选）赔付登记 |

### 4.3 处理操作

| 操作 | 说明 |
|------|------|
| 变更状态 | 待处理→处理中→已处理→已关闭 |
| 添加处理备注 | 追加记录，留痕（操作人+时间），不可删除 |
| 指派处理人 | 「常见逻辑补充」分派给运营成员 |
| 赔付登记 | 「常见逻辑补充」损坏/遗失类工单登记赔付信息（金额/方式/备注），供后续核对 |

**交互**：

| 操作 | 响应 |
|------|------|
| 从订单管理"标记异常"进入 | 打开标记异常弹窗（见页面三）生成工单 |
| 变更状态 | 推进工单状态，记录变更 |
| 添加备注 | 追加留痕 |
| 指派 | 选运营成员分派 |
| 点击关联订单 | 跳订单详情 |
| 关闭 | 关闭面板回列表 |

### 4.4 状态

加载中、正常、已关闭（只读展示不可再操作，可重开由权限定）。

## 五、页面三：标记异常弹窗（从订单管理触发）

### 5.1 目标

运营在订单管理对某订单发起异常标记，生成工单。

### 5.2 字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 关联订单 | 自动带入 | 从订单管理带入订单号 |
| 异常类型 | 必填 | 选第 2.2 类型 |
| 处理备注 | 必填 | 问题描述 |
| 指派处理人 | 选填 | 「常见逻辑补充」 |

### 5.3 交互

| 操作 | 响应 |
|------|------|
| 提交 | 校验必填 → 生成工单（待处理）→ 提示成功 → 可跳工单详情 |
| 取消 | 关闭弹窗 |

## 六、业务规则

- 异常类型含 P0 底线相关（锁柜失败、开柜异常），这些工单为高优先级跟踪；
- 用户端售后申请**自动生成工单**，状态与用户端进度同步；
- 处理留痕作为纠纷证据链，**不可删除，仅可追加**；
- 工单关联订单，双向可跳转（工单→订单详情、订单→关联工单）；
- 权限：管理员与运营员均可处理工单；「常见逻辑补充」关闭/重开、赔付登记等敏感操作可限管理员。

## 七、关联数据对象

异常工单 Ticket、订单 Order。「常见逻辑补充」赔付登记可扩展 Ticket 字段或独立赔付记录。
