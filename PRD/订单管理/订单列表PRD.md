## 一、模块概述

### 1.1 模块目标

让运营无需打开小铁原始后台，即可在游寄后台集中管理全部订单：查询、筛选、查看详情、定位问题订单、发起异常标记。是后台日常运营的核心工作页。

### 1.2 页面构成

| 页面 | 说明 |
|------|------|
| 订单看板（列表页） | 筛选 + 订单表格 + 分页，主页面 |
| 订单详情（抽屉/弹窗） | 点击订单行展开，展示全部字段与状态流转 |

### 1.3 入口

- 侧边导航"订单列表"；
- 工作台待办（待支付/超时未取/退款中）带筛选跳入；
- 异常工单的"关联订单"跳入订单详情。

### 1.4 设计与布局

遵循 `design.md`：后台三段布局，侧边导航"订单列表"高亮；主内容区 = 面包屑 + 标题 + 筛选区 + 数据概览条 + 订单表格 + 分页。视觉、组件、语义色引用 design.md；金额/订单号/时间用 mono 字体。

## 二、订单状态口径（与用户端共用）

订单列表展示的状态使用**本地映射后的状态**（结合小铁 processing/completed + is_overdue 本地加工），与用户端、工作台一致。状态 Badge 用色遵循 design.md 语义色。

| 用户可见状态 | 业务类型 | Badge 色 |
|--------------|----------|----------|
| 待支付 | 寄存/寄送 | warning |
| 寄存中 | 寄存 | success |
| 超时寄存中 | 寄存 | danger |
| 待取件结算 | 寄存 | warning |
| 待转运 / 转运中 | 寄送 | success/info |
| 已送达待取 | 寄送 | warning |
| 已完成 | 寄存/寄送 | ink-muted 灰 |
| 已取消 | 寄存/寄送 | ink-muted 灰 |
| 退款中 | 寄存/寄送 | warning |
| 已退款 | 寄存/寄送 | ink-muted 灰 |
| 异常 | 寄存/寄送 | danger |

## 三、页面一：订单看板（列表页）

### 3.1 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"订单列表" |
| B 筛选区 | 状态、业务类型、来源、柜机、日期范围、关键词搜索 + 查询/重置 |
| C 数据概览条 | 「常见逻辑补充」当前筛选结果的订单数、金额合计 |
| D 订单表格 | 主体，含操作列 |
| E 分页 | 右下角 |
| F 数据时效 | 最近同步时间 |

### 3.2 区块 B：筛选与搜索

| 筛选项 | 选项 | 说明 |
|--------|------|------|
| 订单状态 | 全部 + 第二章各状态 | 支持工作台带入预设筛选 |
| 业务类型 | 全部 / 行李寄存 / 行李寄送 | — |
| 订单来源 | 全部 / 自有小程序 / 小铁小程序 | 区分来源 |
| 柜机 | 全部 / 指定柜机 | 下拉/搜索 |
| 日期范围 | 下单时间范围 | 默认近 7 天 |
| 关键词 | 订单号 / 用户手机号（脱敏匹配） | — |
| 操作 | 查询 / 重置 | — |

**交互**：

| 操作 | 响应 |
|------|------|
| 设置筛选后查询 | 表格按条件刷新 |
| 重置 | 清空筛选回默认 |
| 从工作台带筛选进入 | 自动应用预设筛选（如状态=待支付） |

### 3.3 区块 C：数据概览条

| 组件 | 内容 |
|------|------|
| 订单数 | 当前筛选结果总数 |
| 金额合计 | 当前筛选结果实付合计（元，mono） |

> 「常见逻辑补充」帮助运营快速掌握筛选结果规模，非必需可后置。

### 3.4 区块 D：订单表格

**列字段**：

| 列 | 说明 | 对齐 |
|----|------|------|
| 订单号 | 唯一标识，可复制（mono） | 左 |
| 业务类型 | 行李寄存/行李寄送 | 左 |
| 用户 | 手机号（脱敏 138****8888） | 左 |
| 柜机 | 机柜名称 | 左 |
| 格口 | 格口编号 | 左 |
| 柜型 | 小/中/大 | 左 |
| 订单状态 | 本地映射状态 Badge | 左 |
| 来源 | 自有/小铁 | 左 |
| 金额 | 实付金额（元，mono） | 右 |
| 创建时间 | 下单时间（mono） | 左 |
| 更新时间 | 最近状态更新（mono） | 左 |
| 操作 | 查看详情、标记异常 | 左 |

**表格交互**：

| 操作 | 响应 |
|------|------|
| 点击"查看详情"/订单行 | 打开订单详情抽屉（见页面二） |
| 点击"标记异常" | 打开异常标记弹窗 → 生成/跳转异常工单（见异常工单 PRD） |
| 排序 | 按创建/更新时间、金额排序 |
| hover 行 | bg-hover 高亮 |
| 复制订单号 | 复制到剪贴板 + 提示 |
| 导出 | 「常见逻辑补充」导出当前筛选结果（订单导出可标 V2，一期对账用见对账模块） |

### 3.5 区块 E：分页

右下角，含总条数、页码、每页条数（10/20/50）。

### 3.6 页面状态

| 状态 | 表现 |
|------|------|
| 加载中 | 表格骨架屏 |
| 正常 | 订单列表渲染 |
| 空结果 | "无符合条件的订单"空状态 |
| 数据时效 | 展示最近同步时间 |
| 网络/同步异常 | 错误态 + 重试 |

## 四、页面二：订单详情（抽屉/弹窗）

### 4.1 目标

展示单个订单的全部字段与状态流转，提供标记异常入口。

### 4.2 呈现形式

右侧抽屉（Drawer，宽 480–560px，遵循 design.md）从订单行点击展开；也可用居中弹窗。

### 4.3 内容分组

| 分组 | 字段 |
|------|------|
| 基本信息 | 订单号（可复制）、业务类型、来源、订单状态、创建/更新时间 |
| 用户信息 | 用户手机号（脱敏） |
| 柜机格口 | 机柜名称、格口编号、柜型 |
| 寄存信息（寄存单） | 计费模式（短租/长租）、存放时长、开柜凭证（存储码/取物手机号，脱敏） |
| 寄送信息（寄送单） | 目的地、收件联系信息（脱敏）、备注、预计送达 |
| 费用信息 | 实付金额、超时补费、退款状态与金额 |
| 状态流转 | 状态变更时间线（下单→存入→取件等关键节点） |
| 操作 | 标记异常、（如有）查看关联工单 |

### 4.4 交互

| 操作 | 响应 |
|------|------|
| 打开详情 | 抽屉滑出展示订单全字段 |
| 复制订单号 | 复制 + 提示 |
| 标记异常 | 打开异常标记 → 生成/跳异常工单 |
| 查看关联工单 | 若订单已有工单，跳异常工单详情 |
| 关闭 | 关闭抽屉回列表 |

### 4.5 状态

加载中、正常、异常态订单（状态区 danger 高亮 + 突出标记异常）、退款态（费用区展示退款进度）。

## 五、业务规则

- 状态用本地映射（结合小铁 processing/completed + is_overdue，见 FRD），与用户端/工作台一致；
- 明确区分订单来源（自有/小铁）；
- 金额分转元；手机号、开柜凭证、收件信息脱敏；
- 展示最近同步时间，提示数据时效（小铁数据定时同步存在延迟）；
- **一期后台不直接操作退款/改派**——退款由系统触发（锁柜失败）或经售后工单流程；后台订单列表主要做查看与异常标记；
- 权限：管理员与运营员均可查看订单；「常见逻辑补充」收入敏感字段是否对运营员脱敏由业务定（见待确认）。
