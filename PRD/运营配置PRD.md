## 一、模块概述

### 1.1 模块目标

集中管理运营参数：**价格配置**（计费单一事实来源）、**目的地配置**（行李寄送固定清单）。确保参数可配、变更可追溯、权限受控。

### 1.2 页面构成与权限

| 子页 | 说明 | 权限 |
|------|------|------|
| 价格配置 | 短租/长租分档价、超时费率、退款规则参数 | 仅管理员 |
| 目的地配置 | 行李寄送固定目的地清单 | 仅管理员 |

> **整模块仅管理员可见可操作**。运营员侧边导航不展示"运营配置"，工作台快捷入口也不展示（见工作台 PRD 权限差异）。

### 1.3 设计与布局

遵循 `design.md`：后台三段布局，侧边导航"运营配置"高亮；运营配置下设两个子页（顶部 Tab 切换）。视觉、组件、语义色引用 design.md；金额/数字用 mono 字体。

## 二、子页一：价格配置（重点·计费单一事实来源）

### 2.1 页面目标

配置全站计费规则。**一期为全局统一一套价**（不按柜机/网点差异化，差异化留 V2）。此配置是用户端下单页、收费规则弹窗、订单退款计算、后台对账的**唯一计费依据**，不得在各端硬编码。

### 2.2 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"价格配置" + "生效范围：全局"标识 |
| B 短租价格 | 按柜型三档：单价 + 每24h封顶 |
| C 长租价格 | 按柜型三档：月租 |
| D 超时费率 | 短租超时规则说明 + 长租按柜型日费率 |
| E 退款规则参数 | 短租/长租退款阈值与比例阶梯 |
| F 保存操作 | 保存（二次确认）+ 变更记录入口 |

### 2.3 区块 B：短租价格

| 柜型 | 单价（元/小时） | 每24小时封顶（元） |
|------|----------------|--------------------|
| 小格口 | 2 | 9 |
| 中格口 | 2.5 | 12 |
| 大格口 | 3.5 | 17 |

- 每档为可编辑数字输入（mono），带单位；
- 「常见逻辑补充」输入校验：正数、合理范围、封顶≥单价。

### 2.4 区块 C：长租价格

| 柜型 | 月租（元/月，每月按31天计） |
|------|------------------------------|
| 小格口 | 25 |
| 中格口 | 30 |
| 大格口 | 40 |

- 可编辑数字输入；
- 页面注明"每月按 31 天计"。

### 2.5 区块 D：超时费率
**可配置数值/参数**：

| 参数 | 说明 | 可配 |
|------|------|------|
| 短租超时计费方式 | 按当前格口短租单价计算 | 可配：计费单位（按小时）、不足单位进位规则（不足1小时按1小时） |
| 计费上限 | 上不封顶 | 可配开关（是否封顶 + 封顶值） |

### 2.6 区块 E：退款规则参数
**短租退款算法**

| 参数 | 说明 | 可配 |
|------|------|------|
| 全额退时限 | 下单后 N 分钟内全额退 | 可配（分钟，默认5） |
| 退款算法 | 退款=实付−使用时长费用 | 可配：使用时长费用的计算基准（按短租单价）、费用≥实付时是否不退（可配开关，默认不退） |

**长租退款**

| 参数 | 说明 | 可配 |
|------|------|------|
| 全额退时限 | 下单后 N 分钟内 | 可配（默认5分钟） |
| 整月退费 | 剩余整月按月租整月退 | 可配开关 |
| 不足整月比例阶梯 | ≤1天50% / 1-2天30% / 2-3天10% / 3-31天不退 | **各时段区间与各比例均可配**（可增删阶梯行） |

> 说明：退款比例阶梯做成**可增删行的表格**（每行=时段区间+退款比例%），便于运营调整档位，不写死四档。

### 2.7 区块 F：保存与变更留痕（按确认）

| 操作 | 响应 |
|------|------|
| 点击"保存" | **二次确认弹窗**："价格修改将立即全局生效，影响用户端下单、退款与对账，确认保存？" |
| 确认保存 | 保存并全局生效 + **记录变更留痕**（操作人、时间、变更前后对比） |
| 取消 | 不保存 |
| 查看变更记录 | 打开变更历史（谁在何时把某项从 X 改为 Y） |

**变更留痕字段**：变更时间、操作人、变更项、原值、新值。留痕不可删除，供审计与纠纷追溯。

保存价格时，增加**生效时间选择**：

| 选项 | 说明 |
|------|------|
| 立即生效 | 保存后即时全局生效 |
| 定时生效 | 指定未来某时间点生效；生效前用户端仍用旧价 |

保存流程更新：选择生效方式 → 二次确认（含生效时间提示）→ 保存 → 变更留痕（记录生效时间）。定时生效的配置在生效前可展示"待生效"状态并可取消/修改。

**变更留痕字段增补**：变更时间、操作人、变更项、原值、新值、**生效方式（立即/定时）、生效时间**。

### 2.8 业务规则

- **计费单一事实来源**：本配置是唯一计费依据，用户端下单页、收费规则弹窗、订单退款、对账均引用，禁止硬编码；
- 一期**全局一套价**，不按柜机差异化（V2）；
- 价格需与小铁柜机实际计费对齐，避免实付与展示不一致；
- 保存即全局生效，需二次确认 + 变更留痕；
- 仅管理员可操作。

### 2.9 页面状态

加载中、正常（展示当前配置）、编辑中（有未保存改动提示）、保存二次确认、保存成功、校验错误（非法数值提示）。

## 三、子页二：目的地配置（行李寄送用）

### 3.1 页面目标

维护行李寄送的固定目的地清单，供用户端寄送时选择。

### 3.2 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"目的地配置" + "新增目的地"按钮 |
| B 筛选/搜索 | 按类型/启用状态筛选、按名称搜索 |
| C 目的地列表表格 | 主体 |
| D 新增/编辑弹窗 | 增改目的地 |

### 3.3 列表字段

| 列 | 说明 |
|----|------|
| 目的地名称 | 如"XX 酒店""XX 火车站" |
| 地址 | 详细地址 |
| 类型 | 酒店/车站/景区/其他 |
| 服务费 | 一口价（元，mono） |
| 预计时效 | 预计送达时长 |
| 启用状态 | 启用/停用（Badge） |
| 操作 | 编辑、启用/停用、删除 |

### 3.4 新增/编辑弹窗字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 目的地名称 | 必填 | — |
| 地址 | 必填 | — |
| 类型 | 必填 | 酒店/车站/景区/其他 |
| 服务费 | 必填 | 一口价（元） |
| 预计时效 | 必填 | 如"约4小时内送达" |
| 启用状态 | — | 默认启用 |

### 3.5 交互

| 操作 | 响应 |
|------|------|
| 新增目的地 | 打开弹窗，填写保存 |
| 编辑 | 打开弹窗回填修改 |
| 启用/停用 | 切换状态；**停用后用户端寄送不展示该目的地** |
| 删除 | 二次确认后删除（「常见逻辑补充」建议软删除，避免历史订单丢失关联） |
| 筛选/搜索 | 列表过滤 |

| 操作 | 响应 |
|------|------|
| 删除目的地 | 二次确认后**软删除**——从用户端与可选清单移除，但后台保留记录，历史订单的目的地关联不丢失 |

> 软删除的目的地不出现在用户端寄送选择与后台正常列表，但可通过"已删除"筛选查看，保证历史订单可追溯。

### 3.6 业务规则

- 目的地为用户端行李寄送目的地选择的来源；
- 停用即时对用户端隐藏；
- 服务费为该目的地一口价（与寄送计费一致）；
- 仅管理员可操作。
