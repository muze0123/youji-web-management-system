## 一、页面概述

### 1.1 页面目标

登录页面是后台管理系统的入口，目标是**安全、快速地完成管理员/运营员身份认证**，支持记住账号、密码可见切换等提升体验的功能，登录成功后跳转至工作台。

### 1.2 页面定位与导航

- 后台系统**唯一入口**，未登录状态下所有页面路由均跳转至本页；
- 登录成功后跳转至「工作台」（index.html）；
- 面向管理员与运营员，登录后权限由角色决定（见系统设置-角色管理）；
- 账号体系：后台独立账号（手机号+密码），与用户端小程序账号不共用。

### 1.3 设计与布局

- 遵循 `design.md`：登录页采用**左右分栏布局**（品牌面板 + 登录表单面板）；
- 左侧品牌面板：渐变背景（#0047B3 → #0066FF）+ 装饰球体 + 品牌标识 + 系统文案；
- 右侧登录表单面板：白色背景 + 欢迎标题 + 登录表单 + 辅助链接；
- 视觉、语义色引用 `design.md`，登录按钮采用 primary 色渐变；
- 响应式：桌面端（≥1280px）左右分栏，移动端堆叠显示（品牌面板隐藏，仅显示表单）。

**交互**：纯展示，无交互。

### 3.2 区块 B：登录表单面板

#### 3.2.1 欢迎标题

| 组件 | 内容 | 样式 |
|------|------|------|
| 标题 | "欢迎回来" | ink-title、text-[28px]、font-bold |
| 副标题 | "登录您的管理账号" | ink-sub、text-[15px] |

#### 3.2.2 手机号输入

| 属性 | 说明 |
|------|------|
| 类型 | tel |
| 占位符 | "请输入手机号" |
| 最大长度 | 11 |
| 前置图标 | Lucide phone 图标 |
| 校验规则 | 必填；格式为 1 开头的 11 位手机号 |
| 错误提示 | "请输入手机号" / "请输入有效的手机号" |
| 样式 | h-[52px]、border-line、rounded-lg、input-focus-ring |

**交互**：
- 聚焦时显示蓝色描边（box-shadow: 0 0 0 3px rgba(0,102,255,.15)）；
- blur 时触发校验，错误时显示红色边框与错误提示；
- input 时清除已有错误状态。

#### 3.2.3 密码输入

| 属性 | 说明 |
|------|------|
| 类型 | password（默认）/ text（切换后） |
| 占位符 | "请输入密码" |
| 前置图标 | Lucide lock 图标 |
| 后置按钮 | 眼睛图标（切换密码可见性） |
| 校验规则 | 必填；长度至少 6 个字符 |
| 错误提示 | "请输入密码" / "密码长度至少为6个字符" |
| 样式 | h-[52px]、border-line、rounded-lg、input-focus-ring |

**交互**：
- 聚焦时显示蓝色描边；
- blur 时触发校验；
- 点击眼睛图标切换密码明文/密文；
- input 时清除已有错误状态。

#### 3.2.4 记住我

| 属性 | 说明 |
|------|------|
| 类型 | checkbox |
| 标签 | "记住我" |
| 样式 | w-[16px] h-[16px]、accent-primary |

**交互**：
- 勾选后：登录成功时手机号存入 localStorage（Base64 编码）；
- 下次进入：自动回填手机号并勾选；
- 取消勾选：登录时清除存储；
- 存储键名：`rememberedUser_youji`。

#### 3.2.5 忘记密码

| 属性 | 说明 |
|------|------|
| 类型 | 链接 |
| 文本 | "忘记密码？" |
| 样式 | text-primary、hover:text-primary-hover |

**交互**：
- 点击：弹出 Toast 提示"请联系管理员重置密码"；
- 原型阶段不实现完整重置流程，仅展示提示；
- 「常见逻辑补充」后续可扩展为：发送验证码到注册手机号 → 验证后设置新密码。

#### 3.2.6 登录按钮

| 属性 | 说明 |
|------|------|
| 文本 | "登 录" |
| 样式 | primary 渐变底（#0066FF → #0052CC）、白字、h-[52px]、rounded-lg、font-semibold、box-shadow |
| loading 状态 | 旋转图标 + "登录中..." |

**交互**：
- hover：上浮 2px（transform: translateY(-2px)）、阴影增强；
- active：复位；
- 点击/Enter 提交：前端校验 → 校验通过显示 loading（模拟 2 秒）→ 验证凭证；
- 成功：Toast 绿色提示"登录成功！正在跳转..." → 存入 auth_token → 跳转 index.html；
- 失败：Toast 红色提示"手机号或密码错误" → 恢复按钮状态。

**凭证规则**（mock）：
- 正确账号：手机号 13812345678，密码 123456；
- 错误账号：提示"手机号或密码错误"。

### 3.3 区块 C：Toast 提示

| 属性 | 说明 |
|------|------|
| 位置 | 页面顶部居中（fixed） |
| 样式 | rounded-lg、text-[13px]、font-medium、transition-opacity |
| 类型 | info（默认）、success（登录成功）、error（登录失败） |

## 四、页面状态

| 状态 | 表现 |
|------|------|
| 初始 | 表单空（若有记住我则回填手机号）、按钮可点击 |

| 登录失败 | Toast 错误提示、恢复按钮状态 |

## 五、业务规则

- 登录账号为后台独立账号，与用户端小程序账号不共用；
- 手机号格式校验：1 开头的 11 位数字；
- 密码最小长度：6 位字符；
- 记住我功能：仅存储手机号（Base64 编码），不存储密码；
- 登录成功后：存入 `auth_token_youji`（localStorage），作为后续页面鉴权依据；
- 忘记密码：原型阶段仅提示联系管理员，后续可扩展短信验证码重置流程；
- 安全：密码输入框禁用自动填充建议（autocomplete="current-password"）。
