## 一、页面概述

### 1.1 页面目标

工作台是运营登录后的第一落地页，目标是让运营**一眼看到需要处理的事项**并快速进入处理，同时对当日业务有基本感知。核心定位：**待办驱动为主，关键指标为辅**。

### 1.2 页面定位与导航

- 侧边导航**第一项**，登录后**默认进入**本页；
- 面向管理员与运营员，**权限有差异**（运营员看不到配置类入口与收入，见第五章）；
- 数据来源：订单 Order、异常工单 Ticket、柜点 Cabinet 实时统计。

### 1.3 设计与布局

- 遵循 `design.md`：后台三段布局，侧边导航"工作台"高亮；
- 主内容区从上到下：欢迎/概览条 → 待办事项区（核心）→ 异常告警区 → 快捷入口区 → 简要趋势；
- 视觉、组件、语义色引用 `design.md`；数字用 mono 字体；图表用 Chart.js。

## 三、区块详细规格

### 3.1 区块 A：欢迎/概览条

| 组件 | 内容 | 说明 |
|------|------|------|
| 欢迎语 | "你好，[用户名]" + 角色 + 日期 | 「常见逻辑补充」可加问候 |
| 今日订单 | 今日订单数（mono） | 今日新增订单 |
| 今日收入 | 今日实收金额（元，mono） | **仅管理员可见**（见第五章） |
| 在存订单 | 当前寄存中/在途订单数 | 反映当前业务负载 |
| 数据时效 | 最近更新时间 | 数据来自同步，存在延迟 |

### 3.2 区块 B：待办事项区（核心）

以卡片/列表展示各类待办，每项：待办名 + **数量角标** + 简述 + "去处理"入口。数量为实时统计，点击带筛选跳转对应模块。

| 待办项 | 统计口径 | 点击跳转（带筛选） | 优先级 |
|--------|----------|--------------------|--------|
| 待处理工单 | 状态=待处理 的异常工单数 | → 异常工单页，筛选=待处理 | P0 |
| 锁柜失败待处理 | 支付成功但锁柜失败、待处理的订单/工单 | → 异常工单页，类型=锁柜失败/开柜异常 | P0 |
| 待支付订单 | 状态=待支付 的订单数 | → 订单看板，筛选=待支付 | P1 |
| 超时未取订单 | 状态=超时寄存中 的订单数 | → 订单看板，筛选=超时寄存中 | P1 |
| 退款处理中 | 状态=退款中 的订单数 | → 订单看板，筛选=退款中 | P1 |
| 用户售后待处理 | 来源=用户售后 且待处理的工单 | → 异常工单页，来源=用户售后+待处理 | P1 |

**交互**：

| 操作 | 响应 |
|------|------|
| 点击某待办"去处理"/卡片 | 跳对应模块并带上筛选条件 |
| 数量角标 | 实时数量；为 0 时置灰/弱化，非 0 时高亮（待处理类用 warning/danger 强调） |
| 无任何待办 | 展示"暂无待办，一切正常"友好态 |

**排序/强调规则**：P0 待办（待处理工单、锁柜失败）置顶并用强调色；数量为 0 的待办弱化展示。

### 3.3 区块 C：异常/告警区

展示需关注但非逐单处理的系统级告警：

| 告警项 | 触发 | 展示 |
|--------|------|------|
| 柜机离线 | 有柜机 online=offline | 离线柜机数 + 入口 |
| 异常率偏高 | 当日异常率超阈值 | danger 提示 + 跳数据概览/异常工单 |
| 数据同步异常 | 小铁同步失败/延迟过大 | 「常见逻辑补充」提示同步状态 |

**交互**：点击告警 → 相关模块查看；无告警时该区收起或显示"系统正常"。

### 3.4 区块 D：快捷入口区（按权限）

常用功能快捷入口，宫格/按钮形式：

| 入口 | 目标 | 权限 |
|------|------|------|
| 订单看板 | → 订单管理 | 管理员/运营员 |
| 异常工单 | → 异常工单 | 管理员/运营员 |
| 对账中心 | → 对账中心 | 管理员/运营员 |
| 数据概览 | → 数据概览 | 管理员/运营员 |
| 运营配置 | → 运营配置 | **仅管理员** |

**交互**：点击入口跳对应页；运营员**不展示**运营配置入口（见第五章）。

### 3.5 区块 E：简要趋势

近 7 天业务小趋势，用 Chart.js 呈现，点击跳数据概览看详情。

| 图表 | 说明 |
|------|------|
| 近 7 天单量趋势 | 折线/柱状小图 |
| 近 7 天收入趋势 | 折线/柱状小图（**收入仅管理员可见**） |

**交互**：hover 显示当日数值；点击图表/"查看更多" → 数据概览页。

## 五、权限差异（按确认）

| 内容 | 管理员 | 运营员 |
|------|--------|--------|
| 今日收入 / 收入趋势 | ✅ 可见 | ❌ 不可见（经营收入敏感，运营员隐藏；如业务要求可放开） |
| 待办事项（工单/订单类） | ✅ | ✅ |
| 异常告警 | ✅ | ✅ |
| 快捷入口·运营配置 | ✅ | ❌ 不展示 |
| 其他快捷入口 | ✅ | ✅ |

> 运营员进入工作台时，收入相关数字与"运营配置"入口不展示，其余一致。

## 六、业务规则

- 待办数量为**实时统计**（来自订单、工单数据），点击带筛选跳转对应模块；
- 待办口径与订单看板/异常工单一致（本地映射状态）；
- P0 待办（待处理工单、锁柜失败）为运营每日必处理项，强调展示；
- 数据来自定时同步存在延迟，标注最近更新时间；
- 权限按角色控制（见第五章）。

---

## 七、关联数据对象

订单 Order（各状态计数、今日订单/收入、在存数）、异常工单 Ticket（待处理/售后计数）、柜点 Cabinet（离线告警）。本页仅**读取统计展示**，处理动作在跳转后的模块内完成。
