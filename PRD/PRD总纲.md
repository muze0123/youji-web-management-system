## 一、系统概述

### 1.1 系统目标

游寄后台管理系统是运营团队管理业务的中枢，一期核心目标：

1. **订单可管**：集中查看、筛选、定位全部订单（含用户端自有小程序订单与小铁小程序订单），无需打开小铁原始后台；
2. **异常可控**：标记、流转、处理异常与售后工单，形成责任可追溯的证据链；
3. **财务可对**：留存订单数据、按日汇总、导出与小铁结算单人工核对（一期过渡方案）；
4. **数据可视**：展示单量、收入、使用率、异常率等基础经营指标；
5. **参数可配**：配置价格、目的地、权限等运营参数。

### 1.2 系统边界

- 一期后台**不含**：节点调度（酒店节点/运力/无人车，V2）、完整自动对账结算（V2）、多城市多租户（V3）；
- 后台数据来源：本系统自有订单 + 小铁只读数据接口同步（机柜/柜门/订单）；柜控写操作（锁柜/开柜/凭证）由业务系统触发，后台主要做管理与监控。

### 1.3 使用角色

| 角色 | 权限范围 |
|------|----------|
| 管理员 | 全部功能，含运营配置、权限管理 |
| 运营员 | 订单查看、异常处理、对账查看、数据查看；不含配置与权限管理 |

### 1.4 设计基调（B 端后台）

- 经典后台布局：**左侧导航菜单 + 顶部栏 + 主内容区**；
- 清晰、高信息密度、以表格与筛选为主；
- 中性配色 + 品牌色点缀（青绿/游寄主色用于强调与选中态）；
- 状态用色统一（呼应用户端：进行中青绿、待办/临期橙、异常/超时红、终态灰）。

## 三、模块一：订单看板

### 3.1 页面目标

运营集中查看、筛选、定位全部订单，查看订单详情，并可对订单发起异常标记。是后台使用频率最高的页面。

### 3.2 页面结构

| 区块 | 说明 |
|------|------|
| A 筛选区 | 状态、业务类型、来源、柜机、日期范围、订单号/手机号搜索 |
| B 数据概览条 | 「常见逻辑补充」当前筛选结果的订单数、金额汇总 |
| C 订单列表表格 | 主体，分页 |
| D 订单详情弹窗/抽屉 | 点击行展开 |
| E 数据新鲜度提示 | 最近同步时间 |

### 3.3 区块 A：筛选与搜索

| 筛选项 | 选项 | 说明 |
|--------|------|------|
| 订单状态 | 全部/待支付/寄存中/超时寄存中/待取件结算/已完成/待转运/转运中/已送达待取/已取消/退款中/已退款/异常 | 本地映射状态 |
| 业务类型 | 全部/行李寄存/行李寄送 | — |
| 订单来源 | 全部/自有小程序/小铁小程序 | 区分来源 |
| 柜机 | 按机柜筛选 | 下拉/搜索 |
| 日期范围 | 下单时间范围 | 默认近 7 天 |
| 关键词搜索 | 订单号 / 用户手机号（脱敏匹配） | — |

### 3.4 区块 C：订单列表表格

**列字段**：

| 列 | 说明 |
|----|------|
| 订单号 | 唯一标识，可复制 |
| 业务类型 | 行李寄存/行李寄送 |
| 用户 | 手机号（脱敏） |
| 柜机 | 机柜名称 |
| 格口 | 格口编号 |
| 柜型 | 小/中/大 |
| 订单状态 | 本地映射状态，Badge 用色（含"超时寄存中"） |
| 来源 | 自有/小铁 |
| 金额 | 实付金额（元，分转元展示） |
| 创建时间 | 下单时间 |
| 更新时间 | 最近状态更新 |
| 操作 | 查看详情、标记异常 |

**表格交互**：

| 操作 | 响应 |
|------|------|
| 点击"查看详情" | 打开订单详情弹窗/抽屉 |
| 点击"标记异常" | 打开异常标记（生成/跳转异常工单，见模块二） |
| 排序 | 按创建/更新时间、金额排序 |
| 分页 | 分页加载 |
| 导出 | 「常见逻辑补充」导出当前筛选结果（一期对账用，见模块三；订单导出标 V2 亦可） |

### 3.5 区块 D：订单详情弹窗

展示订单全部字段与状态流转记录：

| 分组 | 字段 |
|------|------|
| 基本信息 | 订单号、业务类型、来源、状态、创建/更新时间 |
| 用户信息 | 用户手机号（脱敏） |
| 柜机格口 | 机柜、格口、柜型 |
| 寄存信息 | 存放时长/计费模式（短租/长租）、开柜凭证（存储码/取物手机号，脱敏） |
| 寄送信息 | 目的地、收件联系信息、备注、预计送达（寄送订单） |
| 费用信息 | 实付金额、超时补费、退款状态与金额 |
| 状态流转 | 状态变更时间线 |
| 操作 | 标记异常、（如涉及）查看关联工单 |

### 3.6 业务规则

- 状态用本地映射（结合小铁 processing/completed + is_overdue 加工出"超时寄存中"等），见 FRD；
- 明确区分订单来源（自有/小铁）；
- 金额分转元；手机号、开柜凭证脱敏；
- 展示最近同步时间，提示数据时效（小铁数据定时同步存在延迟）；
- 一期后台不直接操作退款/改派（退款由系统触发或经售后工单流程），后台主要做查看与异常标记。

### 3.7 关联数据对象

订单 Order、柜点 Cabinet、格口 Locker、取件码 PickupCode、支付记录 Payment、异常工单 Ticket。

### 4.1 页面目标

标记、流转、记录处理异常订单与用户售后申请，形成责任可追溯的证据链。是保障服务可靠的核心运营工具。

### 4.2 页面结构

| 区块 | 说明 |
|------|------|
| A 筛选区 | 工单状态、异常类型、来源、日期、关联订单号搜索 |
| B 工单列表表格 | 主体 |
| C 工单详情/处理面板 | 查看与处理 |

### 4.3 区块 A：筛选

| 筛选项 | 选项 |
|--------|------|
| 工单状态 | 全部/待处理/处理中/已处理/已关闭 |
| 异常类型 | 全部/延误/损坏/遗失/取件码（存储码）错误/支付异常/开柜异常/其他 |
| 来源 | 运营标记 / 用户售后申请 |
| 日期范围 | 创建时间 |
| 搜索 | 工单号 / 关联订单号 |

### 4.4 区块 B：工单列表

| 列 | 说明 |
|----|------|
| 工单号 | 唯一标识 |
| 关联订单 | 订单号，可跳订单详情 |
| 异常类型 | 枚举 |
| 来源 | 运营标记/用户售后 |
| 工单状态 | 待处理/处理中/已处理/已关闭，Badge |
| 处理人 | 当前处理人 |
| 创建时间 | — |
| 更新时间 | — |
| 操作 | 处理、查看 |

### 4.5 区块 C：工单详情/处理面板

| 分组 | 内容 |
|------|------|
| 工单信息 | 工单号、关联订单、异常类型、来源、状态、创建/更新时间 |
| 关联订单摘要 | 订单号、业务类型、柜机、状态（点击进订单详情） |
| 用户诉求 | 用户售后申请的问题描述、图片（若来自用户端售后） |
| 处理记录 | 处理备注留痕（可追加），每条含操作人、时间 |
| 处理操作 | 变更状态（待处理→处理中→已处理→已关闭）、添加备注、指派处理人 |

**交互**：

| 操作 | 响应 |
|------|------|
| 新建/标记异常 | 从订单看板"标记异常"进入，选类型+填备注生成工单 |
| 变更状态 | 按流转推进工单状态 |
| 添加处理备注 | 追加记录，留痕（操作人+时间） |
| 指派处理人 | 「常见逻辑补充」分派给运营成员 |
| 点击关联订单 | → 订单详情 |

### 4.6 业务规则

- 异常类型枚举含开柜异常（呼应用户端取件异常→客服介入）；
- 用户端售后申请自动生成工单，状态与用户端进度同步；
- 处理留痕作为纠纷证据链，不可删除，仅可追加；
- 三条 P0 异常底线（支付成功锁柜失败、取件超时补费、开柜异常）在后台以工单形式跟踪。

### 4.7 关联数据对象

异常工单 Ticket、订单 Order。

## 五、模块三：对账中心（过渡方案）

### 5.1 页面目标

留存并按日汇总订单数据，导出与小铁结算单人工核对，标记差异。一期为过渡方案（数据导出人工核对），完整自动对账为 V2。

### 5.2 页面结构

| 区块 | 说明 |
|------|------|
| A 日期范围选择 | 对账周期 |
| B 按日汇总表 | 每日订单数、金额、退款、差异 |
| C 导出与差异标记 | 导出 CSV、标记差异 |

### 5.3 按日汇总表字段

| 列 | 说明 |
|----|------|
| 对账日期 | 按日 |
| 订单数 | 当日订单总数 |
| 本地金额 | 本系统记录金额（元） |
| 小铁金额 | 小铁结算金额（元） |
| 差异金额 | 本地 − 小铁 |
| 退款金额 | 当日退款合计 |
| 差异状态 | 一致 / 待核查 / 已确认 |
| 操作 | 查看明细、标记差异 |

### 5.4 交互

| 操作 | 响应 |
|------|------|
| 选日期范围 | 汇总展示 |
| 查看某日明细 | 下钻到当日订单明细 |
| 导出 CSV | 导出明细供与小铁结算单人工比对 |
| 标记差异 | 记录差异并留痕，状态置"待核查/已确认" |

### 5.5 业务规则

- 一期采用"数据导出 + 人工核对"过渡方案，完整自动对账依赖与小铁的对账协议条款（待确认）；
- 差异需人工确认，留痕；
- 金额分转元；
- 退款、超时补费金额纳入对账口径（与计费退款规则一致，见行李寄存 PRD 3.4.2）。

### 5.6 关联数据对象

对账记录 Reconcile、订单 Order、支付记录 Payment。

### 6.1 页面目标

展示基础经营指标，支撑运营决策与汇报。一期只做基础指标与简单图表，不做复杂 BI。

|------|------|

| 指标 | 说明 |

| 异常率 | 异常订单 / 总订单 |
| 寄存/寄送结构 | 两类业务占比 |

### 6.4 图表与拆分

| 图表 | 说明 |
|------|------|

### 6.5 业务规则

- 数据来自订单，口径与订单看板一致。

### 6.6 关联数据对象

订单 Order、柜点 Cabinet、格口 Locker。

## 七、模块五：运营配置

### 7.1 页面目标

配置价格、目的地、权限等运营参数。仅管理员可操作。

### 7.2 子模块

| 子模块 | 说明 | 权限 |
|--------|------|------|
| 价格配置 | 短租/长租分档价、超时费率、退款规则参数 | 管理员 |
| 目的地配置 | 行李寄送固定目的地清单 | 管理员 |
| 权限管理 | 后台账号与角色 | 管理员 |

### 7.3 价格配置

**配置项（与行李寄存计费口径一致，见行李寄存 PRD 第二章、3.4.2）**：

| 分组 | 字段 |
|------|------|
| 短租价格 | 按柜型（小/中/大）：单价（元/小时）、每 24h 封顶价 |
| 长租价格 | 按柜型：月租（元/月，每月按 31 天计） |
| 超时费率 | 短租按短租规则；长租按柜型（小 7/中 8/大 9 元/天，可配） |
| 退款规则参数 | 5 分钟全额退阈值、长租不足整月退款比例阶梯（≤1天/1-2天/2-3天/3-31天） |
| 生效范围 | 「常见逻辑补充」按柜机/网点或全局配置 |

**交互**：增改价格项，保存即生效（影响用户端展示与计费、后台对账口径）；「常见逻辑补充」价格变更留痕，避免争议。

> **重要**：价格与退款参数是计费的**单一事实来源**，用户端下单页、收费规则弹窗、订单退款、对账均引用此配置，不得硬编码。

### 7.4 目的地配置

| 字段 | 说明 |
|------|------|
| 目的地名称 | 如"XX 酒店""XX 火车站" |
| 地址 | 详细地址 |
| 类型 | 酒店/车站/景区/其他 |
| 服务费 | 一口价（元） |
| 预计时效 | 预计送达时长 |
| 启用状态 | 启用/停用 |

**交互**：增删改目的地，供用户端行李寄送目的地选择使用；停用后用户端不展示。

### 7.5 权限管理

| 字段 | 说明 |
|------|------|
| 后台账号 | 用户名 |
| 角色 | 管理员/运营员 |
| 状态 | 启用/停用 |
| 操作 | 新增、编辑角色、停用、重置密码 |

**交互**：管理员增删改后台账号与角色；运营员不可见本子模块。

### 7.6 业务规则

- 价格配置需与小铁柜机实际计费对齐，避免实付与展示不一致；
- 配置修改留痕（操作人、时间、变更前后）；
- 仅管理员可操作运营配置与权限管理。

### 7.7 关联数据对象

价格配置、目的地配置、格口 Locker（计费）、订单 Order（目的地）、后台账号/角色。
