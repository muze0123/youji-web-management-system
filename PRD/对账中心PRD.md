## 一、模块概述

### 1.1 模块目标

留存并按日汇总订单与金额数据，将**本地系统金额**与**小铁结算金额**自动比对，发现差异、标记留痕、导出核对，为财务提供对账依据。一期定位为过渡方案，先保证账目可核、差异可查，不做自动追款与结算。

### 1.2 数据来源（按确认）

| 数据 | 来源 |
|------|------|
| 本地金额 | 自有系统的订单 Order / 支付记录 Payment 自动汇总 |
| 小铁金额 | 小铁只读接口自动拉取（订单金额，单位分，时间范围≤7天） |
| 差异 | 系统自动计算：本地 − 小铁 |

### 1.3 页面构成

| 页面 | 说明 |
|------|------|
| 按日对账汇总 | 日期范围 + 按日汇总表 + 导出 |
| 当日明细下钻 | 点某日展开当日订单级明细比对 |
| 差异标记弹窗 | 对差异日/差异单标记与留痕 |

### 1.4 入口与布局

- 侧边导航"对账中心"；
- 遵循 `design.md`：后台三段布局，侧边导航高亮；主内容区 = 面包屑 + 标题 + 日期范围 + 汇总表 + 导出。金额/数字/时间用 mono 字体。

## 二、页面一：按日对账汇总

### 2.1 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"对账中心" + "过渡方案"说明 |
| B 日期范围 | 对账周期选择 + 同步小铁数据按钮 |
| C 汇总概览条 | 「常见逻辑补充」所选周期：总订单、本地合计、小铁合计、差异合计、差异笔数 |
| D 按日汇总表 | 每日一行，含差异状态 |
| E 导出 | 导出 CSV |

### 2.2 区块 B：日期范围

| 组件 | 说明 |
|------|------|
| 日期范围选择 | 起止日期（受小铁接口≤7天约束，见业务规则） |
| 同步小铁数据 | 触发从小铁只读接口拉取该周期金额（「常见逻辑补充」也可定时自动同步，手动为兜底） |
| 最近同步时间 | 展示小铁数据最近拉取时间 |

### 2.3 区块 D：按日汇总表

| 列 | 说明 | 对齐 |
|----|------|------|
| 对账日期 | 按日 | 左 |
| 订单数 | 当日订单总数 | 右 |
| 本地金额 | 本系统记录合计（元，mono） | 右 |
| 小铁金额 | 小铁结算合计（元，mono） | 右 |
| 差异金额 | 本地 − 小铁（元，mono；非0用 danger/warning 强调） | 右 |
| 退款金额 | 当日退款合计（元，mono） | 右 |
| 差异状态 | 一致 / 待核查 / 已确认（Badge） | 左 |
| 操作 | 查看明细、标记差异 | 左 |

**差异状态 Badge**：一致 success、待核查 warning、已确认 ink-muted 灰。

**交互**：

| 操作 | 响应 |
|------|------|
| 选日期范围 + 同步 | 拉取小铁金额、汇总展示 |
| 点"查看明细" | 下钻当日订单级明细（页面二） |
| 点"标记差异" | 打开差异标记弹窗（页面三） |
| 差异金额非0 | 用色强调，提示需核查 |

### 2.4 区块 E：导出

| 操作 | 响应 |
|------|------|
| 导出 CSV | 导出当前周期汇总（或明细）供财务与小铁结算单人工比对 |

### 2.5 页面状态

加载中、正常、无数据、小铁数据未同步（提示先同步）、同步失败（错误态+重试）。

## 三、页面二：当日明细下钻

### 3.1 目标

点某日展开该日**订单级**明细比对，定位具体差异在哪几笔。

### 3.2 明细表字段

| 列 | 说明 |
|----|------|
| 订单号 | mono，可跳订单详情 |
| 业务类型 | 寄存/寄送 |
| 来源 | 自有/小铁 |
| 本地金额 | 元，mono |
| 小铁金额 | 元，mono（无匹配时标"缺失"） |
| 差异 | 本地−小铁，非0强调 |
| 状态 | 一致/差异 |

### 3.3 交互

| 操作 | 响应 |
|------|------|
| 点订单号 | 跳订单详情 |
| 筛选"仅看差异" | 只显示有差异的订单 |
| 标记差异 | 对某笔标记与留痕 |
| 返回 | 回按日汇总 |

### 3.4 差异类型（辅助识别）

| 类型 | 说明 |
|------|------|
| 金额不一致 | 本地与小铁金额不等 |
| 本地有小铁无 | 小铁未查到对应订单 |
| 小铁有本地无 | 本地未记录（需排查） |

> 「常见逻辑补充」订单匹配依赖可关联标识（见待确认），无标识则按时间+金额+柜机辅助匹配。

## 四、页面三：差异标记弹窗

### 4.1 目标

对差异日或差异单标记核查结论并留痕。

### 4.2 字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 差异对象 | 自动带入 | 某日/某订单 |
| 差异金额 | 自动带入 | — |
| 核查结论 | 必填 | 待核查 / 已确认（差异原因归类） |
| 差异原因 | 必填 | 如：退款时间差、同步延迟、漏单、金额口径差异等 |
| 备注 | 选填 | 详细说明 |

### 4.3 交互

| 操作 | 响应 |
|------|------|
| 提交标记 | 保存核查结论 + 留痕（操作人、时间、结论、原因）→ 差异状态更新 |
| 取消 | 关闭 |

**留痕不可删除**，供财务审计。

## 五、业务规则

- 一期**过渡方案**：本地自动汇总 + 小铁接口自动拉取金额 + 自动算差异，运营人工核查标记，**只对账不结算**（追款/清结算 V2）；
- **小铁接口约束**：金额单位分（展示转元）、时间范围≤7天——**日期范围跨度受此限制**，超范围需分段同步（「常见逻辑补充」系统可分段轮询汇总）；
- 差异 = 本地金额 − 小铁金额，非0需人工核查；
- 差异状态：一致/待核查/已确认，标记留痕不可删；
- 金额口径与订单管理/支付一致，含退款、超时补费；
- 订单匹配依赖可关联标识（待确认）；
- 权限：管理员与运营员均可查看对账（「常见逻辑补充」标记差异等操作是否限管理员由业务定）。
