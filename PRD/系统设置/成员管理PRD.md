# 成员管理 PRD

**文档版本**：v1.0
**页面名称**：成员管理（Member Management）
**所属模块**：系统设置 / 成员管理
**文档状态**：待评审
**创建日期**：2026-07-05

---

## 1. 页面概述

成员管理页面用于管理企业组织架构及成员信息，支持新增、编辑、删除成员，以及批量导入、导出、调岗、删除等批量操作。左侧展示可交互的组织架构树，右侧展示选中节点的成员列表与筛选工具。

**访问权限**：需拥有「成员管理 - 查看」功能权限，无权限时菜单隐藏，不可访问。

---

## 2. 页面整体布局

页面整体为**左右双栏布局**，左右两栏各自独立滚动，高度撑满页面可视区域。

```
┌─────────────────────┬──────────────────────────────────────────────────┐
│                     │                                                  │
│   左侧组织架构树     │              右侧成员列表区                       │
│   （固定宽度）       │         （占据剩余宽度）                          │
│                     │                                                  │
└─────────────────────┴──────────────────────────────────────────────────┘
```

## 3. 左侧组织架构树

### 3.1 整体布局

```
┌──────────────────────┐
│  杭州渡河科技有限公司  │  ← 根节点（公司名称）
│  （10人）            │
│  ▼ 部门名称1（5人）  │  ← 一级部门（展开状态）
│    ▼ 部门名称2（3人）│  ← 二级部门（选中，高亮）  [...]
│      ▶ 部门名称3（1人）│  ← 三级部门（收起状态）
└──────────────────────┘
```

### 3.2 节点结构说明

| 元素 | 说明 |
|------|------|
| 展开/收起图标 | `▶` 表示收起，`▼` 表示展开，点击切换折叠状态 |
| 节点名称 | 部门名称或公司名称 |
| 人数标注 | 显示「(N人)」，N = 本节点直属人数 + 所有子部门人数之和 |
| 选中态 | 当前选中节点高亮显示 |
| `...` 操作菜单 | 鼠标 hover 节点时在节点右侧出现，点击展开操作菜单 |

### 3.3 节点的 `...` 操作菜单

**根节点（公司名称）的菜单选项：**

| 选项 | 触发行为 |
|------|---------|
| 编辑部门 | 弹出「编辑部门」弹窗，仅可编辑公司名称与负责人 |
| 添加子部门 | 弹出「添加子部门」弹窗 |

> 根节点不支持删除，菜单中无「删除部门」选项。

**非根节点（各级子部门）的菜单选项：**

| 选项 | 触发行为 |
|------|---------|
| 编辑部门 | 弹出「编辑部门」弹窗 |
| 添加子部门 | 弹出「添加子部门」弹窗，默认上级部门为当前节点 |
| 删除部门 | 见 3.6 节删除流程 |

### 3.4 点击节点行为

| 点击对象 | 右侧区域表现 |
|---------|------------|
| 根节点（公司名称）| 展示全公司所有成员 |
| 任意部门节点 | 展示该部门成员（根据右侧「数据范围」筛选项决定是否含子部门成员）|

默认进入页面时，默认选中根节点（公司名称）。

### 3.5 编辑部门弹窗

**弹窗标题**：「编辑部门」

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 部门名称 | 是 | 文本输入，最多 20 字；不可修改上级部门 |
| 负责人 | 否 | 下拉多选，支持输入成员名称关键词模糊搜索；最多可选 10 人；已选负责人全显示在下拉框内，支持单独删除（点击已选项旁 `×`）|

**交互规则：**

- 点击【确定】：校验必填项
  - 部门名称为空：字段下方红色提示「请输入部门名称」
  - 部门名称与同级部门重名：字段下方红色提示「部门名称已存在」
  - 校验通过：关闭弹窗，Toast 提示「编辑部门成功」，左侧树节点名称同步更新
- 点击【取消】或右上角 `×`：关闭弹窗，不保存

### 3.6 添加子部门弹窗

**弹窗标题**：「添加子部门」

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 部门名称 | 是 | 文本输入，最多 20 字 |
| 上级部门 | 是 | 只读，回显当前操作的节点名称，不可修改 |
| 负责人 | 否 | 同「编辑部门」弹窗，最多选 10 人 |

**交互规则：**

- 点击【确定】：校验必填项
  - 部门名称为空：字段下方红色提示「请输入部门名称」
  - 部门名称与同级部门重名：字段下方红色提示「部门名称已存在」
  - 校验通过：关闭弹窗，Toast 提示「添加成功」，左侧树新增对应子节点
- 点击【取消】或 `×`：关闭弹窗，不保存

### 3.7 删除部门

**触发方式**：点击非根节点 `...` 菜单中的「删除部门」。

**情况一：部门下有成员或子部门**

弹出「提示」弹窗：
- 内容：「需要先删除部门门下的员工，再删除该部门」
- 按钮：【我知道了】，点击关闭弹窗，不执行删除

**情况二：部门无成员且无子部门**

弹出确认弹窗：
- 标题：「确认删除该部门吗？」
- 按钮：【取消】【确定】
- 点击【确定】：关闭弹窗，Toast 提示「删除成功」，左侧树移除该节点
- 点击【取消】：关闭弹窗，不执行删除

> **说明**：组织架构人数 = 当前部门人数 + 子部门人数（所有层级之和）

## 4. 右侧成员列表区

### 4.1 整体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  部门名称2  部门人数 2  总人数 3                                  │  ← 标题区
├──────────────────────────────────────────────────────────────────┤
│  搜索成员：[输入框]  角色：[下拉]  状态：[下拉]  数据范围：[下拉] │  ← 筛选区
│                                             [查询]  [重置]       │
├──────────────────────────────────────────────────────────────────┤
│  [批量操作▾]                                         [添加成员]  │  ← 工具栏
├──────────────────────────────────────────────────────────────────┤
│  ☐  成员名称  所属角色  手机号  所在部门  添加日期  入职时间/工龄 │  ← 表头
│                              最近修改时间  状态  操作             │
├──────────────────────────────────────────────────────────────────┤
│  数据行 × N                                                      │
├──────────────────────────────────────────────────────────────────┤
│  分页控件                                                        │
└──────────────────────────────────────────────────────────────────┘
```

### 4.2 标题区

格式：`[部门名称]  部门人数 N  总人数 M`

| 字段 | 说明 |
|------|------|
| 部门名称 | 当前左侧选中节点的名称；根节点时显示公司名称 |
| 部门人数 | 当前部门的直属成员数 |
| 总人数 | 当前部门 + 所有子部门的成员数之和 |

## 5. 筛选区

### 5.1 筛选控件说明

| 控件 | 类型 | 说明 |
|------|------|------|
| 搜索成员 | 输入框 | placeholder「请输入成员名称/昵称/手机号搜索」；成员名称模糊搜索，手机号精准搜索 |
| 角色 | 下拉单选 | 展示系统中所有角色，默认空（不筛选）；选择后筛选该角色下的成员 |
| 状态 | 下拉单选 | 选项：全部（默认）/ 启用 / 禁用 |
| 数据范围 | 下拉单选 | 选项：仅展示部门的直属成员（默认）/ 展示部门及子部门的成员 |

### 5.2 筛选交互规则

| 操作 | 行为 |
|------|------|
| 点击【查询】或按回车 | 应用所有筛选条件，列表刷新至第 1 页 |
| 点击【重置】 | 所有筛选项恢复默认值，列表刷新至第 1 页 |
| 各筛选条件之间 | AND 关系，同时生效 |
| 切换左侧组织树节点 | 筛选条件保持不变，列表数据以新节点重新加载 |

## 6. 工具栏

### 6.1 【批量操作】下拉按钮

点击展开下拉菜单，包含五个选项：

| 操作 | 说明 |
|------|------|
| 批量跟进 | 详见 6.2 节 |
| 批量导入 | 详见 6.3 节 |
| 批量导出 | 详见 6.4 节 |
| 批量删除 | 详见 6.5 节 |
| 批量调岗 | 详见 6.6 节 |

### 6.2 批量跟进

- 未选中成员时点击：Toast 提示「请选择要跟进的成员」
- 已选中成员时点击：弹出「选择跟进人」弹窗，选择跟进 BD 后点击【确定】执行跟进，Toast 提示「跟进成功」

### 6.3 批量导入

点击后弹出「导入数据」弹窗。

**弹窗布局：**

```
┌─────────────────────────────────────────┐
│ 导入数据                              × │
├─────────────────────────────────────────┤
│                                         │
│        [📁 拖放区域]                    │
│   点击将要导入的文件拖拽到这里上传        │
│              [上传文件]                 │
│                                         │
│ 注意事项：                              │
│ 1. 请按照导入规范的头行字段顺序导入      │
│ 2. 建议每次导入数量不超过规定字数        │
│ 3. 相同学生，上系统已有数据为准          │
│ 4. 有任何疑问，「任务单-导入记录」中...  │
│                                         │
│ 下载导入模板                            │
├─────────────────────────────────────────┤
│                        [取消]  [导入]   │
└─────────────────────────────────────────┘
```

**导入模板字段（去除昵称、国家、固定资产后）：**

| 字段名称 | 是否必填 | 输入方式 | 校验提示 |
|---------|---------|---------|---------|
| 成员名称 | 是 | 文本输入，最多 30 字 | 超字数标红提示 |
| 手机号 | 是 | 数值型，11 位 | 非法格式标红提示 |
| 所属部门 | 是 | 下拉选择，支持关键词模糊搜索 | 选择已存在部门 |
| 入职时间 | 否 | 日期格式 YYYY-MM-DD | 非法格式标红提示 |
| 备注信息 | 否 | 文本，最多 600 字 | — |

**导入流程：**

```
上传文件 → 填写表格数据 → 上传文件 → 文件格式/表头校验
    ↓ 校验失败
点击【另存】重新上传文件
    ↓ 校验通过
数据逐行校验 → 生成成员入口 → 导入成员 → 导入完成
                    ↓ 数据有误
               标红错误行，支持在线修改后重新导入
```

**导入结果处理：**

- 全部成功：Toast 提示「导入成功，导入N名，点击查看」，点击「查看」跳转至「任务中心 - 导入记录」页面
- 部分成功/失败：弹出结果详情，展示成功条数与失败原因，支持下载失败数据

**导入结果文件字段（展示当前时间戳命名）：**
`成员管理导入_YYYYMMDDHHMMSS.xlsx`

### 6.4 批量导出

点击后弹出「导出提示」弹窗：

- 提示内容：「本次将导出当前筛选条件下的所有成员数据，结果可在「任务中心 - 下载任务」中查看和下载。」
- 按钮：【取消】【确认导出】
- 点击【确认导出】：提交导出任务，Toast 提示「导出任务已提交，请前往任务中心查看」
- 导出文件命名：`成员管理导出_YYYYMMDDHHMMSS.xlsx`
- 导出字段（去除昵称、国家、固定资产）：成员名称、所属角色、手机号、所在部门、入职时间、工龄、添加日期、最近修改时间、状态

### 6.5 批量删除

- 未选中成员时点击：Toast 提示「请选择要删除的成员」
- 已选中成员时点击：弹出确认弹窗
  - 标题：「确认删除所选成员吗？」
  - 内容：「删除后，所选成员账号将被注销，相关数据将被保留。」
  - 按钮：【取消】【确定】
  - 点击【确定】：执行删除，Toast 提示「删除成功」，列表刷新
  - 点击【取消】：关闭弹窗，不执行删除

### 6.6 批量调岗

- 未选中成员时点击：Toast 提示「请选择要调岗的成员」
- 已选中成员时点击：弹出「批量调岗」弹窗

**批量调岗弹窗：**

```
┌────────────────────────────────────────┐
│ 批量调岗                             × │
├────────────────────────────────────────┤
│ * 目标部门：                           │
│  [根目录（企业名称）               ▾] │
│    ▼ 部门名称1                        │
│    ▼ 部门名称2  ← 当前选中项（高亮）  │
```

- 点击【取消】：关闭弹窗

### 6.7 【添加成员】按钮

点击后弹出「添加成员」抽屉弹窗，详见第 8 节。

## 7. 成员列表

### 7.1 列表字段说明

| 列名 | 说明 |
|------|------|
| 复选框 | 表头复选框为全选/全不选当前页；选中行高亮 |
| 成员名称 | 圆形头像 + 成员姓名；主账号（企业注册账号）在头像右上角显示专属标识图标 |
| 所属角色 | 显示成员所属角色名称；有多个角色时以「、」分隔展示 |
| 手机号 | 完整手机号（不做脱敏处理） |
| 所在部门 | 成员所在部门名称 |
| 添加日期 | 格式 `YYYY-MM-DD` |
| 入职时间/工龄 | 格式：`YYYY-MM-DD \| N天`；「N天」为从入职时间到当前日期的天数；未填写入职时间则显示「——」 |
| 最近修改时间 | 格式 `YYYY-MM-DD HH:mm:ss` |
| 状态 | Toggle 开关，见 7.2 节 |
| 操作 | 编辑、删除，见 7.3 节 |

### 7.2 状态 Toggle

| 状态 | 含义 | 切换行为 |
|------|------|---------|
| 开启（蓝色）| 成员账号启用，可正常登录 | 点击后立即切换为关闭态，Toast 提示「已禁用」，无需二次确认 |
| 关闭（灰色）| 成员账号禁用，无法登录 | 点击后立即切换为开启态，Toast 提示「已启用」，无需二次确认 |

**特殊规则：**
- 主账号（企业注册账号）状态 Toggle 置灰不可操作，始终保持启用状态
- 禁用账号后，该成员当前已登录的会话立即失效

### 7.3 操作列

| 操作 | 触发行为 |
|------|---------|
| 编辑 | 打开「编辑成员」抽屉弹窗，预填当前成员信息，详见第 8 节 |
| 删除 | 弹出确认弹窗：标题「确认删除该成员吗？」，按钮【取消】【确定】；点击【确定】删除，Toast 提示「删除成功」；主账号不显示「删除」入口 |

**主账号操作列特殊规则：**
- 不显示「删除」操作
- 「编辑」操作正常显示

### 7.4 列表空状态

| 场景 | 展示内容 |
|------|---------|
| 当前节点无成员（未搜索）| 空状态图 + 「暂无成员数据」|
| 筛选/搜索后无结果 | 空状态图 + 「未搜索到相关结果」|

### 7.5 分页

- 默认每页 10 条，支持切换：10 / 20 / 50 条/页
- 格式：页码导航 + 每页条数选择 + 跳至 N 页 + 共 N 条记录 第 X/Y 页

## 8. 添加/编辑成员弹窗

添加与编辑使用同一抽屉弹窗，字段和校验逻辑一致。以下统称「成员编辑弹窗」。

- **添加成员**：弹窗标题「添加成员」，所有字段为空/默认值
- **编辑成员**：弹窗标题「编辑成员」，各字段预填当前成员数据

### 8.1 弹窗字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 成员名称 | 是 | 文本输入，最多 30 字 |
| 所属部门 | 是 | 下拉树形选择，支持关键词模糊搜索，单选；默认带入当前左侧选中的部门 |
| 手机号 | 是 | 11 位数字；输入后点击【绑定】与子账号手机绑定；编辑时若修改手机号需重新绑定 |
| 所属角色 | 否 | 下拉多选，展示系统中所有角色 |
| 入职时间 | 否 | 日期选择器，格式 `YYYY-MM-DD` |
| 登录密码 | 是（仅添加时）| 6-20 字符，须包含数字；编辑时不显示此字段（密码通过独立入口修改）|
| 备注信息 | 否 | 文本输入，最多 600 字；右下角实时显示「已输入字数/600」|

### 8.2 手机号绑定交互

1. 输入手机号后，点击字段右侧【绑定】按钮
2. 系统向该手机号发送验证码，输入框切换为验证码输入框
3. 填写验证码后完成绑定，该手机号成为该成员子账号的登录手机号
4. 绑定成功后手机号字段显示已绑定状态，【绑定】按钮变为【解绑】

### 8.3 所属部门 — 树形下拉

- 点击下拉展开组织架构树
- 根目录（企业名称）可选
- 选中某部门后：
  - 若在「添加子部门」场景中打开：重新选择时只能选当前部门同级或上级部门，不能选下级部门
  - 普通添加成员场景：可选任意部门
- 支持关键词搜索过滤部门名称
- 单选，选中后下拉收起并回显所选部门名称

### 8.4 校验规则

| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 成员名称 | 不可为空 | 「请输入成员名称」 |
| 成员名称 | 最多 30 字 | 输入框限制，无法继续输入 |
| 所属部门 | 不可为空 | 「请选择所属部门」 |
| 手机号 | 不可为空 | 「请输入手机号」 |
| 手机号 | 必须为 11 位数字 | 「请输入正确的手机号」 |
| 手机号 | 系统内不可重复 | 「手机号已存在」 |
| 登录密码 | 不可为空（仅添加）| 「请输入登录密码」 |
| 登录密码 | 6-20 字符，含数字 | 「密码须为 6-20 位且包含数字」 |
| 备注信息 | 最多 600 字 | 输入框限制，无法继续输入 |

### 8.5 提交行为

**点击【确定】：**
- 触发全量字段校验
- 校验不通过：对应字段下方显示红色提示，弹窗不关闭
- 校验通过：提交接口
  - **添加成功**：关闭弹窗，弹出「子账号信息」成功弹窗（见 8.6 节）
  - **编辑成功**：关闭弹窗，Toast 提示「编辑成功」，列表刷新
  - **接口失败**：Toast 提示具体错误信息，弹窗不关闭

**点击【取消】或右上角 `×`：**
- 关闭弹窗，所有修改不保存

添加成功后弹出子账号信息确认弹窗：

## 9. 默认态与搜索态

### 9.1 默认态

- 进入页面时，左侧默认选中根节点（公司名称），右侧展示全公司所有成员
- 筛选区所有控件为默认值（搜索框空，角色不筛选，状态为「全部」，数据范围为「仅展示部门的直属成员」）

### 9.2 搜索态

- 任意筛选条件有值时进入搜索态
- 列表数据根据筛选条件过滤展示
- 筛选条件之间为 AND 关系
- 切换左侧节点后，搜索条件保持，以新节点 + 当前筛选条件重新请求数据

## 11. 数据说明

| 数据项 | 规则 |
|-------|------|
| 部门人数 | 当前部门直属成员数，不含子部门成员 |
| 总人数 | 当前部门 + 所有子部门成员总和 |
| 组织架构树节点人数标注 | 同总人数规则（含所有下级子部门人数）|
| 工龄（天）| 以入职日期到当前系统日期计算天数；未填入职时间时工龄显示「——」 |
| 主账号标识 | 企业注册账号，在头像右上角显示专属图标标识 |
| 最近修改时间 | 每次编辑成员信息成功后由后端自动更新 |
| 添加日期 | 成员被创建加入系统的日期，创建后不可更改 |
| 导出文件时间戳 | 以实际操作时间生成，格式 `YYYYMMDDHHMMSS` |

## 12. 异常场景处理

| 场景 | 表现 | 处理方案 |
|------|------|---------|
| 左侧组织树加载失败 | 树区域显示「加载失败，点击重试」占位 | 点击重新请求组织树接口 |
| 右侧成员列表加载失败 | 列表区显示「加载失败，点击重试」占位 | 点击重新请求成员列表接口 |
| 添加/编辑成员接口失败 | Toast 提示接口返回的错误信息，弹窗不关闭 | — |
| 删除成员接口失败 | Toast 提示「操作失败，请稍后重试」 | — |
| 状态切换接口失败 | Toggle 恢复到切换前状态，Toast 提示「操作失败，请稍后重试」 | — |
| 导入文件格式错误 | 弹窗内提示「文件格式不支持，请上传 .xlsx/.xls 文件」 | 重新上传文件 |
| 导入文件表头错误 | 弹窗内提示「文件表头不匹配，请下载最新模板重新填写」 | 点击「下载导入模板」重新操作 |
| 批量调岗接口失败 | Toast 提示「操作失败，请稍后重试」，成员部门不变 | — |
| 网络断开执行任何写操作 | Toast 提示「网络异常，请检查网络后重试」 | — |

## 13. 交互规则汇总

| 规则 | 说明 |
|------|------|
| 主账号不可删除 | 操作列不显示「删除」入口，批量删除时跳过主账号 |
| 主账号 Toggle 不可操作 | 状态列 Toggle 置灰，始终保持启用 |
| 根节点不支持删除 | `...` 菜单中无「删除部门」选项 |
| 有成员/子部门的部门不可删除 | 弹出提示引导先清空部门 |
| 状态切换无需确认 | 直接切换，立即生效 |
| 删除成员直接二次确认 | 无其他前置限制 |
| 批量操作未选中时给出提示 | Toast 提示「请先选择要操作的成员」 |
| 筛选切换节点后保留筛选条件 | 切换左侧部门节点不重置筛选项 |
| 数据范围「仅展示直属成员」为默认 | 进入页面及每次切换节点时默认该选项 |
| 导入模板字段不含昵称、国家、固定资产 | 已从模板和导入流程中去除上述字段 |
| 成员列表不显示昵称、国家、固定资产 | 已从表格列中去除上述字段 |
| 添加/编辑成员表单不含昵称、国家、固定资产 | 已从表单字段中去除上述字段 |
