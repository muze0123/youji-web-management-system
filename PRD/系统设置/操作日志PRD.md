**文档状态**：待评审
**创建日期**：2026-07-10

---

## 1. 页面概述

操作日志是后台管理系统的全量操作审计功能，自动记录所有管理员在后台执行的关键操作行为，提供完整的操作溯源能力，支持问题排查、合规审计与异常行为发现。

**核心特性：**
- 覆盖后台全部模块的所有操作类型；
- 记录操作前后数据对比，可追溯每次变更的具体内容；
- 数据永久保留，只读不可删；
- 通过角色管理的数据权限控制可见范围（超级管理员可查看全员日志，普通角色可配置仅查看本人日志）；
- 支持多维度筛选与 Excel 导出。

## 3. 权限控制

操作日志的**可见数据范围**由「角色管理」中该角色的**数据权限**字段控制：

| 数据权限配置 | 可查看范围 |
|------------|---------|
| 全部数据 | 可查看全员所有操作日志 |
| 本部门及下级部门 | 可查看本部门及下属部门成员的操作日志 |
| 本部门 | 可查看本部门成员的操作日志 |
| 仅本人 | 只能查看自己的操作日志 |

> 超级管理员默认拥有「全部数据」权限，可查看所有人的日志。

**页面访问权限**：需拥有「操作日志 - 查看」功能权限，无权限时菜单隐藏。

## 4. 筛选区

### 4.1 筛选控件

| 控件 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| 操作时间 | 日期区间选择器 | 近 7 天 | 精确到年月日；最大可查询范围不限（永久保留）|
| 操作人 | 下拉单选 + 模糊搜索 | 全部（权限范围内）| 下拉列表仅展示当前数据权限范围内的成员 |
| 操作模块 | 下拉单选 | 全部 | 见 4.2 节模块枚举 |
| 操作类型 | 下拉单选 | 全部 | 见 4.3 节类型枚举；选择操作模块后，操作类型自动过滤为该模块支持的类型 |
| 操作对象 | 输入框 | 空 | 支持输入订单号、用户手机号、优惠券名称等关键词，模糊搜索 |

### 4.2 操作模块枚举

### 4.3 操作类型枚举

| 操作类型值 | 展示名称 | 适用模块 |

| 按钮 | 位置 | 行为 |
|------|------|------|

## 5. 默认态

**进入页面时的默认状态：**
- 操作时间：近 7 天
- 其余筛选项：全部/空
- 列表按操作时间**倒序**排列（最新操作在最上方）

## 6. 日志列表

### 6.1 表格列定义

| 列名 | 宽度 | 说明 |
|------|------|------|
| 操作时间 | `160px` | 格式 `YYYY-MM-DD HH:mm:ss`，精确到秒；支持点击列头切换升/降序 |
| 操作人 | `100px` | 操作员工姓名 |
| 操作模块 | `120px` | 如「优惠券管理」「成员管理」|
| 操作类型 | `100px` | 如「编辑」「状态变更」「删除」|
| 操作对象 | `180px` | 操作的目标实体，见 6.2 节 |
| 操作摘要 | `auto` | 简短描述本次操作内容，超长省略，hover 展示完整 Tooltip |
| 操作 | `80px` | 「查看详情」链接 |

### 6.2 操作对象字段展示规则

| 操作模块 | 操作对象展示内容 |
|---------|--------------|
| 账号登录 | IP 地址 + 设备信息（如「Chrome / Mac」）|
| 行李寄存订单 | 订单号（如 `LS202507100001`）|
| 行李寄送订单 | 订单号（如 `LD202507100001`）|
| 异常工单 | 工单号（如 `AW202507100001`）|
| 柜机管理 | 柜机名称（如「龙翔桥地铁站A口·柜机#03」）|
| 运营配置 | 配置项名称（如「计费规则」「营业时间」）|
| 优惠券管理 | 券名称（如「新用户立减5元」）+ 批次号（如有）|
| 成员管理 | 成员姓名 + 手机号（脱敏）|
| 角色管理 | 角色名称（如「BD负责人」）|
| 系统设置 | 设置项名称 |

### 6.3 操作摘要生成规则

操作摘要由系统自动生成，格式为：`{操作人} {操作类型} 了 {操作对象}（{核心变更内容摘要}）`

**示例：**

| 操作类型 | 示例摘要 |
|---------|---------|
| 登录 | 「张三 登录了后台管理系统（IP: 192.168.1.1）」|
| 新增 | 「张三 新增了 优惠券【新用户立减5元】」|
| 编辑 | 「张三 编辑了 成员【李四】（修改了角色、所属部门）」|
| 删除 | 「张三 删除了 角色【BD负责人】」|
| 状态变更 | 「张三 变更了 订单【LS202507100001】状态（寄存中 → 已完成）」|
| 导入 | 「张三 导入了 成员数据（共100条，成功98条，失败2条）」|
| 导出 | 「张三 导出了 行李寄存订单数据（共500条）」|
| 配置变更 | 「张三 修改了 计费规则（短租小柜单价 ¥2/h → ¥2.5/h）」|
| 发放优惠券 | 「张三 发放了 优惠券【新用户立减5元】给 50 位用户」|
| 登出 | 「张三 登出了后台管理系统」|

### 6.4 列表分页

- 默认每页 20 条；可切换 20 / 50 / 100 条/页
- 格式：页码导航 + 每页条数选择 + 跳页输入框 + 共N条 第X/Y页

### 7.3 基本信息字段

| 字段 | 说明 |
|------|------|
| 操作时间 | 精确到秒，格式 `YYYY-MM-DD HH:mm:ss` |

| 操作 IP | 发起操作时的客户端 IP 地址 |
| 操作模块 | 所属业务模块名称 |
| 操作类型 | 操作动作名称 |
| 操作对象 | 操作目标实体的标识信息 |

### 7.4 变更详情展示规则

根据操作类型，变更详情展示方式不同：

**① 编辑 / 配置变更**

以三列表格形式展示所有变更字段：

| 字段名称 | 变更前 | 变更后 |
|---------|-------|-------|
| 发放总量 | 100 | 200 |
| 状态 | 停用 | 启用 |

- 仅展示**发生变更的字段**，未变更字段不展示
- 变更前/后均为空时，展示「—」
- 变更后值以**橙色**标注，以便与变更前区分

**② 新增**

以字段列表展示新增记录的所有字段值：

| 字段名称 | 值 |
|---------|---|
| 券名称 | 新用户立减5元 |
| 券种 | 立减券 |
| 面额 | ¥5.00 |
| 使用门槛 | 满¥20可用 |
| 适用范围 | 通用 |
| 有效期 | 领取后30天 |
| 发放总量 | 100 |
| 状态 | 启用 |

**③ 删除**

以字段列表展示被删除记录删除前的数据快照（保留现场）。

**④ 状态变更**

```
变更前状态：[寄存中]  →  变更后状态：[已完成]
```

**⑤ 登录 / 登出**

```

```

**⑦ 导出**

```

导出记录数：500 条

**⑧ 发放优惠券**

发放用户数：50 人
每人数量：1 张
发放备注：客服补偿

```

## 9. 导出功能

### 9.1 触发方式

点击筛选区右上角【导出日志】按钮。

### 9.2 导出规则

| 规则 | 说明 |
|------|------|
| 导出范围 | 导出**当前筛选条件**下的全量日志（不受分页限制）|
| 数据权限 | 导出数据仍遵循当前用户的数据权限范围，不可导出无权限的日志 |
| 文件格式 | Excel（.xlsx）|
| 文件命名 | `操作日志导出_{YYYYMMDDHHMMSS}.xlsx` |
| 导出方式 | 异步任务，提交后在「任务列表 - 下载任务」中查看和下载 |

### 9.3 导出文件字段

| 字段 | 说明 |
|------|------|
| 操作时间 | `YYYY-MM-DD HH:mm:ss` |
| 操作人 | 员工姓名 |
| 操作人手机号 | 完整手机号 |
| 操作 IP | 客户端 IP |
| 操作模块 | 模块名称 |
| 操作类型 | 操作动作 |
| 操作对象 | 目标实体标识 |
| 操作摘要 | 完整操作摘要文本 |
| 变更详情 | JSON 格式字符串（含变更前/变更后字段值）|

### 9.4 导出交互

| 操作 | 行为 |
|------|------|
| 点击【导出数据】| 弹出导出确认弹窗：「将导出当前筛选条件下共 N 条日志记录，导出结果可在「任务列表 - 下载任务」中查看」；按钮【确认导出】【取消】|
| 点击【确认导出】| 提交导出任务，Toast「导出任务已提交，请前往任务中心查看」|
| 点击【取消】| 关闭弹窗，不触发导出 |

## 12. 异常场景处理

| 场景 | 表现 | 处理方案 |
|------|------|---------|
| 日志列表加载失败 | 列表区显示「加载失败，点击重试」| 点击重新请求接口 |
| 日志详情加载失败 | 抽屉内显示「加载失败，点击重试」| 点击重新请求 |
| 导出记录数超大（> 10万条）| 弹出提醒「当前筛选条件下共 N 条记录，导出较大可能需要较长时间，确认继续？」| 用户确认后继续提交 |
| 跳页超出范围 | Toast「请输入有效页码」| — |
| 导出任务提交失败 | Toast「导出任务提交失败，请稍后重试」| — |
| 网络断开时操作 | Toast「网络异常，请检查网络后重试」| — |

## 13. 交互规则汇总

| 规则 | 说明 |
|------|------|
| 日志只读不可删 | 页面无删除/清空入口；接口层也应拒绝删除请求 |
| 数据永久保留 | 不设数据清理任务，历史日志永久可查 |
| 数据权限隔离 | 操作人下拉仅展示权限范围内成员；列表数据严格按数据权限过滤 |
| 操作类型联动模块 | 选择操作模块后，操作类型下拉自动过滤为该模块支持的类型 |
| 列表默认时间范围近7天 | 避免首次进入加载数据过多 |
| 列表按操作时间倒序 | 最新操作优先展示 |
| 变更详情仅展示有变化的字段 | 新增/删除操作展示全字段快照 |
| 变更后值橙色标注 | 帮助审计人员快速识别变化点 |
| 只读操作不记录日志 | 如：查看列表、查看详情、搜索等纯浏览行为不写入日志 |
| 导出遵循数据权限 | 不可通过导出绕过数据权限访问限制 |

## 14. 数据说明

| 数据项 | 规则 |
|-------|------|
| 日志写入时机 | 后端操作接口成功返回后写入，操作失败的请求不记录 |
| 操作 IP | 后端从请求 Header 中获取（X-Forwarded-For / RemoteAddr）|
| 变更详情存储格式 | JSON 格式存储变更前后数据，前端解析后以表格展示 |
| 数据保留 | 永久保留，不设 TTL |
| 操作时间精度 | 精确到秒，存储 UTC 时间，前端按本地时区展示 |
| 日志 ID | 系统自动生成，唯一，不对外展示（仅用于详情接口请求）|
| 敏感信息处理 | 密码等敏感字段的变更，变更前/变更后均显示「***」，不记录明文 |
| 手机号展示 | 操作人手机号在列表中不展示；详情和导出中展示完整号码（需有相应权限）|
