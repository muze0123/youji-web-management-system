**文档版本**：v1.0

**所属模块**：系统设置 / 角色管理  
**文档状态**：待评审  
**创建日期**：2026-07-05  

---

## 1. 页面概述

角色管理页面用于管理企业内部的权限角色，支持新增、编辑、删除自定义角色，并为每个角色配置功能权限与数据权限范围、管理角色成员。

**访问权限**：需拥有「角色管理 - 查看」功能权限，无权限时菜单隐藏，不可访问。

---

## 2. 页面整体布局

页面整体为**左右双栏布局**，左右两栏高度撑满页面可视区域，各自独立滚动。

```
┌───────────────────┬──────────────────────────────────────────────┐
│                   │                                              │
│   左侧角色列表栏   │              右侧角色详情区                   │
│   （固定宽度）     │         （占据剩余宽度）                      │
│                   │                                              │
└───────────────────┴──────────────────────────────────────────────┘
```

### 2.1 左侧角色列表栏

从上到下依次为：

```
┌──────────────────────┐
│  [ 🔍 搜索角色 ]      │  ← 搜索框
├──────────────────────┤
│  超级管理员           │  ← 角色列表（可滚动）
│  管理员              │
│  BD负责人            │
│  角色名称            │
│  …                   │
├──────────────────────┤
│  [ 新增角色 ]        │  ← 底部固定按钮
└──────────────────────┘
```

- 搜索框固定在顶部
- 角色列表区域独立滚动
- 【新增角色】按钮固定在底部，不随列表滚动

### 2.2 右侧角色详情区

从上到下依次为：

```
┌──────────────────────────────────────────────────────────────────┐
│  角色名称（N人）                          [编辑]  [删除]（条件显示）│  ← 标题行

├──────────────────────────────────────────────────────────────────┤
│  [角色成员]  [功能权限]  [数据权限]                               │  ← Tab 栏
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Tab 内容区（可滚动）                                            │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 3. 左侧角色列表

### 3.1 搜索框

- placeholder：「搜索角色」
- 支持根据角色名称关键词模糊搜索
- 搜索无结果时，列表区显示「未搜索到相关结果」占位图（通用空状态图）
- 搜索框内容清空后，恢复完整角色列表

### 3.2 角色列表

**排序规则：**

| 位置 | 角色类型 | 排序逻辑 |
|------|---------|---------|
| 始终置顶 | 超级管理员 | 不参与排序，固定第一位 |
| 置顶之后 | 其余角色 | 按创建时间倒序排列 |

**拖拽排序：**

- 支持对「超级管理员」以外的角色进行拖拽排序
- 超级管理员不可被拖动，始终固定在第一位
- 拖拽过程中，被拖动项有视觉拖拽态（跟随鼠标浮起）
- 松手后排序立即生效并持久化保存
- 超级管理员位置不受其他角色拖拽影响

**选中态：**

- 进入角色管理页时，默认选中列表第一项（超级管理员），右侧展示对应详情
- 点击列表中任意角色，该行高亮选中，右侧切换为对应角色详情
- 当前选中项始终保持高亮

### 3.3 【新增角色】按钮

- 固定在左侧列表底部
- 点击后弹出「新增角色」抽屉弹窗（详见第 6 节）
- 创建成功后，新角色插入列表顶部（超级管理员下方），并自动选中该角色，右侧展示新角色详情

## 4. 右侧角色详情区

### 4.1 标题行

**内容：**`角色名称（N人）`

- N 为当前角色的成员总数
- 标题右侧操作按钮区域（见下表）

**操作按钮显示规则：**

| 角色类型 | 【编辑】 | 【删除】 |
|---------|---------|---------|
| 超级管理员 | 不显示 | 不显示 |
| 其他所有角色 | 显示 | 显示 |

### 4.2 元信息行

从左到右依次展示以下字段，各字段之间等间距排列：

| 字段 | 说明 |
|------|------|
| 状态 | `已启用` 或 `已禁用` |
| 创建人 | 创建该角色的成员姓名；超级管理员的创建人显示「系统」 |
| 创建时间 | 精确到年月日时分秒，格式 `YYYY-MM-DD HH:mm:ss` |
| 备注 | 显示备注内容；超级管理员备注显示「系统自动生成」；无备注时显示 `——` |

- 备注内容超出单行时，行尾截断显示 `…`，鼠标 hover 时 Tooltip 全量展示
- 超级管理员角色：创建人为「系统」，创建时间为企业注册时间，备注为「系统自动生成」，不支持【编辑】【删除】

### 4.3 Tab 栏

三个 Tab 依次为：`角色成员` | `功能权限` | `数据权限`

- 默认激活「角色成员」Tab
- 切换 Tab 时，内容区更新，不重置成员搜索框内容

## 5. 角色成员 Tab

### 5.1 整体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  [🔍 请输入成员名称/昵称/手机号搜索]        [移除成员]  [添加成员]  │  ← 工具栏
├──────────────────────────────────────────────────────────────────┤
│  ☐  成员名称   昵称   手机号   所在部门   操作                    │  ← 表头（灰底）
├──────────────────────────────────────────────────────────────────┤
│  ☐  [头像] 张三  XXX  139...  部门名称   ——（或移除）            │
│  ☐  [头像] 李四  XXX  139...  部门名称   移除                    │
│  …                                                               │
├──────────────────────────────────────────────────────────────────┤
│  分页控件                                                        │
└──────────────────────────────────────────────────────────────────┘
```

### 5.2 搜索框

- placeholder：「请输入成员名称/昵称/手机号搜索」
- 支持：成员名称（关键词模糊搜索）、昵称（关键词模糊搜索）、手机号（关键词精准搜索）
- 实时搜索或回车触发（防抖 300ms）
- 搜索无结果时，列表区显示「未搜索到相关结果」空状态占位图

### 5.3 成员列表字段

| 列名 | 说明 |
|------|------|
| 复选框 | 用于批量选择，表头复选框为全选/全不选 |
| 成员名称 | 成员头像（圆形）+ 成员名称；主账号（企业注册账号）在头像右上角显示专属标识图标以区分 |
| 昵称 | 成员昵称 |
| 手机号 | 完整手机号 |
| 所在部门 | 成员所在部门名称 |
| 操作 | 见下方说明 |

**操作列规则：**

| 成员类型 | 操作列展示 |
|---------|----------|
| 主账号（超级管理员角色下）| `——`（不可移除，操作列显示破折号）|
| 普通成员 | 显示「移除」文字链接 |

### 5.4 【移除成员】按钮（批量）

**无选中时：**
- 点击【移除成员】按钮，Toast 提示「请选择要移除的成员」，不弹窗

**有选中时：**
- 点击【移除成员】弹出二次确认弹窗：
  - 标题：「确认将所选成员移出角色吗？」
  - 内容：「删除后，所选成员将不再有【XXX角色名称】角色的相关权限」
  - 按钮：【取消】【确定】
- 点击【确定】：
  - 后端实时校验每个被选中成员删除此角色后是否还有其他角色
  - 若某成员无其他角色：Toast 提示「移除失败，成员至少拥有一个角色」，弹窗关闭，列表不变
  - 若所有选中成员均有其他角色：关闭弹窗，Toast 提示「移除成功」，列表刷新
  - 若部分成员有其他角色、部分无其他角色：仅移除有其他角色的成员，无其他角色的成员保留，Toast 提示「X人移除成功，X人移除失败（成员至少拥有一个角色）」
- 点击【取消】：关闭弹窗，列表不变

### 5.5 单行「移除」操作

- 点击某行「移除」链接，弹出同上二次确认弹窗（内容一致）
- 点击【确定】后，后端实时查询该成员是否还有其他角色：
  - 有其他角色：移除成功，Toast 提示「移除成功」，该行从列表消失
  - 无其他角色：Toast 提示「移除失败，成员至少拥有一个角色」，弹窗关闭，列表不变

### 5.6 【添加成员】按钮

点击后弹出「选择成员」弹窗：

**弹窗布局：**

```
┌─────────────────────────────────────────────────────────────────┐
│ 选择成员                                                  ×     │
├──────────────────────────────┬──────────────────────────────────┤
│  [🔍 请输入部门名称/成员名   │  已选（N/1000）          [清空]  │
│     称/手机号搜索]           │                                  │
│                              │  👤 王琦                    ×   │
│  ☐ XXX集团           ∨     │  👤 胡金玉                   ×   │
│    ☐ XXX公司         ∨     │                                  │
│      ☐ XXX分公司0    >      │                                  │
│      ☐ XXX分公司     ∨     │                                  │
│        ☐ 👤 王琦           │                                  │
│        ☐ 👤 王琦           │                                  │
│    ☐ XXX公司2        >      │                                  │
│                              │                                  │
└──────────────────────────────┴──────────────────────────────────┘
│                                           [取消]  [确定]        │
└─────────────────────────────────────────────────────────────────┘
```

**选择成员弹窗交互规则：**

| 功能 | 说明 |
|------|------|
| 搜索 | 支持输入部门名称、成员名称、手机号搜索；结果实时过滤 |
| 树形结构 | 展示企业完整组织架构；支持展开/收起；默认收起到第一级 |
| 多选 | 支持跨部门多选成员；勾选部门则选中该部门下所有成员 |
| 上限 | 最多可选 1000 人，右侧「已选（N/1000）」实时显示当前已选数量 |
| 右侧已选区 | 实时展示已选成员列表；点击成员右侧 `×` 取消该成员的选中 |
| 【清空】 | 点击后清空所有已选成员 |
| 已在角色中的成员 | 已是该角色成员的人员，在树形列表中置灰且不可选 |
| 【确定】按钮状态 | 未选择任何成员时置灰禁用；有选中成员时可点击 |
| 提交 | 点击【确定】关闭弹窗，Toast 提示「添加成员成功」，成员列表刷新 |
| 取消 | 点击【取消】或 `×` 关闭弹窗，不保存操作 |

### 5.7 分页

- 默认每页 10 条，支持切换每页条数
- 展示格式：页码导航 + 每页条数选择 + 跳至 N 页 + 共 N 条记录 第 X/Y 页

## 8. 新增 / 编辑角色弹窗

- **编辑**：点击右侧详情区【编辑】按钮触发，标题为「编辑角色」，各字段预填当前角色数据

### 8.2 角色信息区块

**角色名称**

| 属性 | 说明 |
|------|------|
| 必填 | 是，左侧标红星 `*` |
| 字数限制 | 最多 20 字，右侧实时显示「已输入字数/20」 |
| placeholder | 「请输入」 |
| 校验时机 | 点击【保存】时校验 |
| 校验规则 | 不可为空；不可与已有角色名称重复（接口校验） |
| 错误提示 | 字段下方红色文字提示，如「请输入角色名称」「角色名称已存在」 |

**角色状态**

| 属性 | 说明 |
|------|------|
| 必填 | 是 |
| 控件 | 单选，选项：`启用` / `禁用` |
| 默认值 | 新增时默认选中「启用」 |

**备注**

| 属性 | 说明 |
|------|------|
| 必填 | 否 |
| 控件 | 多行文本框 |
| 字数限制 | 最多 100 字，右下角实时显示「已输入字数/100」 |
| placeholder | 「请补充说明角色用途」 |

### 8.3 数据权限区块

**区块标题**：「数据权限」+ 「?」图标（hover 展示数据权限说明 Tooltip：「设置该角色可以在各个页面查看数据的范围。仅针对达人级别的用户包含"达人运营人员"等等»，才会生效」）

**数据权限下拉**

| 属性 | 说明 |
|------|------|
| 必填 | 是，左侧标红星 `*` |
| 控件 | 下拉单选 |
| 选项 | 全部数据 / 本部门及下级部门 / 本部门 / 仅本人 |
| placeholder | 「请选择」 |
| 默认值 | 新增时不预选，需用户主动选择 |

### 8.4 功能权限区块

**区块标题**：「功能权限」+ 「?」图标（hover 展示 Tooltip：「设置该角色可以在各个页面查看数据和操作。仅针对达人 运营人员的用户类型包含达人运营'等' 才会生效」）

**全部展开/收起**

- 点击「▶ 全部展开」：展开所有一级菜单及其子项，按钮文字变为「▼ 全部收起」
- 点击「▼ 全部收起」：收起所有展开的项，只显示一级菜单，按钮文字变为「▶ 全部展开」

**「所有权限」总开关**

- 位置：标题行右侧，独立复选框
- 状态说明：

| 显示状态 | 含义 |
|---------|------|
| 复选框空（☐）| 未勾选任何权限 |
| 复选框半选（—）| 已勾选部分权限 |
| 复选框全选（☑）| 已勾选全部权限 |

- 点击「☐ 所有权限」（未选状态）→ 全选所有权限，所有复选框变为 ☑
- 点击「☑ 所有权限」（全选状态）→ 取消所有权限，所有复选框变为 ☐

**树形权限选择交互**

权限树共三级：一级菜单 → 二级菜单 → 三级操作权限

| 操作 | 行为 |
|------|------|
| 勾选一级菜单 | 该菜单下所有子项（二级、三级）全部勾选 |
| 取消一级菜单 | 该菜单下所有子项全部取消 |
| 勾选二级菜单 | 该二级菜单下所有三级操作权限全部勾选；父级一级菜单变为半选 |
| 取消二级菜单 | 该二级菜单下所有三级操作权限全部取消；若父级其他子项有选中则父级保持半选 |
| 单独勾选/取消三级权限 | 子级状态影响父级显示：全选→父级☑，部分选→父级半选（—），全不选→父级☐ |
| 点击一级菜单折叠图标 ▶ | 展开该菜单下的子项 |
| 点击一级菜单折叠图标 ▼ | 收起该菜单下的子项 |

**权限联动说明：**
- 在功能权限中，仅勾选「查看」权限时，用户只能查看页面信息无功能权限
- 若未勾选「查看」权限，但勾选了菜单模块下其他功能权限，则用户依旧可以查看页面信息（隐含查看权限）

### 8.5 底部操作栏（固定在弹窗底部）

**【取消】按钮**：关闭弹窗，所有修改不保存

**【保存】按钮**：

- 触发字段校验（必填项 + 格式校验）
- 校验不通过：对应字段下方显示红色错误提示，弹窗不关闭
- 校验通过：提交接口
  - 成功：关闭弹窗，Toast 提示「新增角色成功」或「编辑成功」
  - 失败：Toast 提示具体错误信息，弹窗不关闭

**校验规则汇总：**

| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 角色名称 | 不可为空 | 「请输入角色名称」 |
| 角色名称 | 最多 20 字 | 输入框限制，无法继续输入 |
| 角色名称 | 不可与已有角色重复 | 「角色名称已存在」 |
| 数据权限 | 不可为空 | 「请选择数据权限」 |
| 角色状态 | 不可为空（默认已选，通常不触发）| — |

## 9. 删除角色

### 9.1 触发方式

点击右侧详情区【删除】按钮。

### 9.2 删除流程

**若角色内有成员：**

弹出「提示」弹窗：
- 标题：「提示」
- 内容：「删除角色前，请先移除角色内的所有成员」
- 按钮：【我知道了】
- 点击【我知道了】关闭弹窗，不执行删除

**若角色内无成员：**

弹出确认弹窗：
- 标题：「确认删除该角色吗？」
- 按钮：【取消】【确定】
- 点击【取消】：关闭弹窗，不执行删除
- 点击【确定】：关闭弹窗，执行删除，Toast 提示「删除成功」
  - 删除成功后，左侧列表移除该角色
  - 右侧自动切换展示列表中的第一个角色（超级管理员）

## 13. 交互规则汇总

| 规则 | 说明 |
|------|------|
| 超级管理员不可编辑/删除 | 右侧详情区不显示【编辑】【删除】按钮 |
| 超级管理员置顶且不可拖动 | 始终排列在角色列表第一位 |
| 其他角色支持拖拽排序 | 拖拽后立即持久化保存 |
| 角色列表无数量上限 | 暂不限制创建角色数量 |
| 成员至少拥有一个角色 | 移除时实时查询，若无其他角色则禁止移除并给出提示 |
| 主账号不可移除（超级管理员角色下）| 操作列显示 `——`，无移除入口 |
| 删除角色前需先移除所有成员 | 有成员时点击删除仅弹出提示，不允许直接删除 |
| 编辑角色弹窗与新增角色弹窗交互一致 | 复用同一套表单校验逻辑 |
| 搜索无结果 | 展示通用空状态占位图，提示文字「未搜索到相关结果」 |
| 功能权限 Tab 内无权限 | 展示空状态提示「暂无功能权限」 |
