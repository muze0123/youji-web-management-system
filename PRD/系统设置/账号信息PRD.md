**文档版本**：v1.0
**页面名称**：账号信息（Account Info）
**所属模块**：系统设置 / 账号信息
**文档状态**：待评审
**创建日期**：2026-07-05

---

## 1. 页面概述

账号信息页面用于展示当前登录成员的个人资料与安全设置，支持修改基本信息、修改登录密码、更换绑定手机号，以及查看最近登录记录。

**访问入口**：
- 侧边栏：系统设置 → 账号信息
- 顶栏：用户图标按钮

**访问权限**：所有已登录成员均可访问本页面查看和修改自己的账号信息。

## 3. Tab 导航

### 3.1 Tab 结构

| Tab | 图标 | 说明 |
|-----|------|------|
| 基本信息 | user | 展示头像、成员资料、快捷操作入口 |
| 安全设置 | shield | 展示密码管理、手机绑定、登录记录 |

### 3.2 Tab 交互规则

| 操作 | 行为 |
|------|------|
| 点击 Tab 标签 | 切换对应内容区域，选中态显示 primary 色底部边框 |
| 默认选中 | 进入页面默认展示「基本信息」Tab |
| 互斥规则 | 两个 Tab 互斥，同一时间仅一个处于选中态 |

## 4. 基本信息 Tab

### 4.1 头像区

| 元素 | 说明 |
|------|------|
| 头像 | 88px 圆形，浅灰底色 + 默认 User 图标占位 |
| hover 态 | 显示半透明黑色遮罩 + 相机图标 |
| 点击 | Toast 提示「头像上传功能开发中，敬请期待」（原型阶段不实现上传） |
| 边框 | 2px line-light 色描边 |

### 4.2 快捷摘要

头像右侧展示三行信息：

| 行 | 内容 | 样式 |
|----|------|------|
| 第1行 | 成员姓名 | 18px / 600 / ink-title |
| 第2行 | `成员ID：` + 成员ID | 13px / ink-sub / mono 字体 |
| 第3行 | 角色 Badge 列表 | 12px / primary 字 + primary-bg 底 / 4px 圆角 |

### 4.3 信息详情列表

每条信息为 label-value 行，label 右对齐（120px 宽），字段间用 line-lighter 色分隔线。

| 字段 | 说明 | 字体 |
|------|------|------|
| 成员名称 | 当前成员的姓名 | sans |
| 成员ID | 系统分配的唯一成员标识 | mono |
| 企业信息 | 所属企业全称 | sans |
| 企业ID | 企业唯一标识 | mono |
| 所属角色 | 成员被分配的角色，多个以「、」分隔 | sans |
| 入职时间 | `YYYY-MM-DD` + 工龄 N 天（自动计算） | sans（日期部分 mono） |
| 绑定手机号 | 脱敏显示（如 `139****0001`） | mono |
| 登录密码 | 掩码显示（`···········`），不可查看明文 | mono |

> **工龄计算规则**：从入职日期到当前系统日期的天数差。原型阶段通过前端 JS 计算。

### 4.4 操作按钮

| 按钮 | 类型 | 触发行为 |
|------|------|---------|
| 修改信息 | primary 主按钮 | 弹出「修改账号信息」弹窗（见 §7.1） |
| 修改密码 | default 次按钮 | 弹出「修改密码」弹窗（见 §7.2） |

## 5. 安全设置 Tab

### 5.1 登录密码区

独立卡片区块（bg-page 浅灰底、line 边框），展示：

| 信息 | 说明 |
|------|------|
| 密码强度 | 显示「强 / 中 / 弱」，颜色对应用 success / warning / danger |
| 上次修改时间 | 格式 `YYYY-MM-DD HH:mm`，mono 字体 |

右侧按钮：

| 按钮 | 类型 | 触发行为 |
|------|------|---------|
| 修改密码 | default 次按钮 | 弹出「修改密码」弹窗（与基本信息 Tab 的修改密码按钮行为一致） |

> 基本信息 Tab 和安全设置 Tab 均提供修改密码入口，两个入口打开同一个弹窗，行为完全一致。

### 5.2 绑定手机号区

独立卡片区块（bg-page 浅灰底、line 边框），展示：

| 信息 | 说明 |
|------|------|
| 当前绑定 | 脱敏手机号（如 `139****0001`），mono 字体 |
| 说明文字 | 「手机号用于登录验证和重要操作确认」，12px ink-muted |

右侧按钮：

| 按钮 | 类型 | 触发行为 |
|------|------|---------|
| 更换绑定 | default 次按钮 | 弹出「更换绑定手机号」弹窗（见 §7.3） |

### 5.3 最近登录记录

| 属性 | 说明 |
|------|------|
| 标题 | 「最近登录记录」 |
| 说明 | 「仅显示最近 10 条」，12px ink-muted |
| 展示形式 | 标准数据表格（`.data-table`） |
| 数据条数 | 固定展示最近 10 条 |

**表格列：**

| 列名 | 说明 | 字体 |
|------|------|------|
| 登录时间 | 格式 `YYYY-MM-DD HH:mm:ss` | mono |
| IP 地址 | 登录来源 IP | mono |
| 设备 / 浏览器 | 如 `Chrome 128 / macOS` | sans |
| 登录结果 | Badge 形式展示 | sans |

**登录结果 Badge 规则：**

| 结果 | Badge 样式 | 示例文案 |
|------|-----------|---------|
| 成功 | success 字 + success-bg 底 | 登录成功、扫码登录 |
| 失败 | danger 字 + danger-bg 底 | 密码错误 |

## 6. 弹窗通用规范

所有弹窗遵循 `design.md §5.6` Modal 规范：

| 属性 | 值 |
|------|-----|
| 遮罩 | `rgba(0,0,0,.3)`，点击遮罩关闭弹窗 |
| 圆角 | 8px |
| 入场动画 | 遮罩淡入 0.15s + 弹窗 slide-in 0.22s |
| 结构 | 标题栏（16px/600 + 关闭 ×）+ 内容区 + 底部按钮区（分隔线，按钮居右） |
| 关闭 × | 20px，ink-muted 色，hover 变 ink-body |
| 表单字段 | label 110px 右对齐 + 控件 max-width 400px / 32px 高 / 4px 圆角 / line 边框 |
| 必填标识 | label 末尾红色 `*`（danger 色） |
| 校验错误 | 控件描边转 danger 色 + 下方 12px danger 字提示 |

## 7. 弹窗详情

### 7.1 修改账号信息弹窗

**弹窗标题**：「修改账号信息」

**弹窗宽度**：760px

**字段说明：**

| 字段 | 必填 | 控件类型 | 说明 |
|------|------|---------|------|
| 成员名称 | 是 | 文本输入 | 最多 30 字符，预填当前姓名 |
| 所属角色 | 否 | 下拉单选 | 选项列表见下方；预填当前角色 |
| 绑定手机号 | 否 | 文本输入 | 11 位数字，mono 字体；修改后原号自动解绑 |

**角色选项列表**：超级管理员、运营主管、财务专员、普通成员、客服专员、设备管理员

**手机号字段提示**：字段下方灰色小字「修改手机号后需重新验证，原手机号将自动解绑。」

**校验规则：**

| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 成员名称 | 不可为空 | 「请输入成员名称（2-30个字符）」 |
| 成员名称 | 2-30 个字符 | 同上 |
| 绑定手机号 | 若填写，须为 1 开头的 11 位数字 | 「请输入正确的11位手机号」 |

**提交行为：**

- 点击【保存】：执行全量校验
  - 校验通过 → 更新 mock data → 关闭弹窗 → 页面重新渲染 → Toast 提示「账号信息修改成功」
  - 校验不通过 → 对应字段标红 + 显示错误提示，弹窗不关闭
- 点击【取消】或右上角 ×：关闭弹窗，所有修改不保存，表单重置

### 7.2 修改密码弹窗

**弹窗标题**：「修改密码」

**弹窗宽度**：560px

> 原型阶段不实际修改密码，仅模拟交互流程。表单提交后直接关闭弹窗 + Toast 提示成功。

**字段说明：**

| 字段 | 必填 | 控件类型 | 说明 |
|------|------|---------|------|
| 当前密码 | 是 | 密码输入框 | placeholder「请输入当前密码」，mono 字体，圆点掩码 |
| 新密码 | 是 | 密码输入框 | placeholder「8-32位，含大小写字母+数字」 |
| 确认密码 | 是 | 密码输入框 | placeholder「请再次输入新密码」 |

**校验规则：**

| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 当前密码 | 不可为空 | 「请输入当前密码」 |
| 新密码 | 8-32 位，必须同时包含大写字母、小写字母和数字 | 「密码需8-32位，且包含大小写字母和数字」 |
| 确认密码 | 必须与新密码完全一致 | 「两次输入的密码不一致」 |

**提交行为：**

- 点击【确认修改】：执行三级校验
  - 校验通过 → 关闭弹窗 → Toast 提示「密码修改成功，下次登录请使用新密码」
  - 校验不通过 → 对应字段标红 + 显示错误提示，弹窗不关闭
- 点击【取消】或 ×：关闭弹窗，表单重置

### 7.3 更换绑定手机号弹窗

**弹窗标题**：「更换绑定手机号」

**弹窗宽度**：560px

> 原型阶段不实际发送短信，验证码为演示模式（输入任意 6 位数字即可通过）。

**字段说明：**

| 字段 | 必填 | 控件类型 | 说明 |
|------|------|---------|------|
| 当前手机号 | — | 只读文本 | 脱敏显示（如 `139****0001`），mono 字体，不可编辑 |
| 新手机号 | 是 | 文本输入 | 11 位数字，placeholder「请输入新手机号」 |
| 验证码 | 是 | 文本输入 + 获取按钮 | 6 位数字，右侧「获取验证码」按钮 |

**验证码获取按钮交互：**

| 状态 | 按钮文案 | 行为 |
|------|---------|------|
| 默认 | 「获取验证码」 | 可点击 |
| 点击后 | 「60s 后重发」→ 倒计时 | 按钮禁用，每秒更新倒计时数字 |
| 倒计时结束 | 恢复「获取验证码」 | 按钮恢复可点击 |
| 点击时 | Toast 提示「验证码已发送（演示模式，输入任意6位数字即可）」 | — |

**校验规则：**

| 字段 | 校验规则 | 错误提示 |
|------|---------|---------|
| 新手机号 | 不可为空 | 「请输入正确的11位手机号」 |
| 新手机号 | 1 开头的 11 位数字 | 同上 |
| 验证码 | 不可为空 | 「请输入6位验证码」 |
| 验证码 | 6 位数字（原型阶段接受任意 6 位数字） | 同上 |

**提交行为：**

- 点击【确认更换】：执行校验
  - 校验通过 → 更新 mock data → 新手机号脱敏处理 → 关闭弹窗 → 页面重新渲染 → Toast 提示「手机号更换成功」
  - 校验不通过 → 对应字段标红 + 显示错误提示，弹窗不关闭
- 点击【取消】或 ×：关闭弹窗，表单重置，清除倒计时

## 8. 数据说明

| 数据项 | 规则 |
|-------|------|
| 成员ID | 系统分配的唯一数字标识 |
| 企业ID | 企业唯一数字标识 |
| 工龄（天） | 入职日期到当前系统日期的天数差，由前端 JS 实时计算 |
| 手机号 | 详情列表和卡片区显示脱敏格式（如 `139****0001`），弹窗内编辑时显示/输入完整号码 |
| 密码 | 详情列表以掩码展示，不可复制或查看明文 |
| 密码强度 | 根据规则判定：强（含大小写+数字+特殊字符）、中（含两种字符类型）、弱（单一类型） |
| 登录记录 | 固定展示最近 10 条，分页逻辑不在原型中实现 |
| 角色列表 | 系统预设角色，原型阶段以静态数组维护 |

## 9. 异常场景处理

| 场景 | 表现 | 处理方案 |
|------|------|---------|
| 修改信息弹窗校验失败 | 对应字段标红 + 下方红色错误提示 | 弹窗不关闭，用户修正后可再次提交 |
| 修改密码弹窗校验失败 | 同上 | 同上 |
| 更换手机号弹窗校验失败 | 同上 | 同上 |
| 验证码倒计时中关闭弹窗 | 倒计时被清除 | 重新打开弹窗时倒计时重置 |
| 头像点击 | Toast 提示「头像上传功能开发中，敬请期待」 | 原型阶段不实现上传 |

## 10. 交互规则汇总

| 规则 | 说明 |
|------|------|
| 默认展示基本信息 Tab | 进入页面时基本信息 Tab 为选中态 |
| 两个密码修改入口行为一致 | 基本信息 Tab 和安全设置 Tab 均打开同一个密码修改弹窗 |
| 手机号脱敏显示 | 详情页和卡片区统一显示 `前3 + **** + 后4` 格式 |
| 手机号修改后自动解绑原号 | 弹窗内提示「修改手机号后需重新验证，原手机号将自动解绑」 |
| 修改信息后页面实时刷新 | mock data 更新后直接调用渲染函数，无需手动刷新页面 |
| 密码掩码不可见 | 登录密码以 `···········` 展示，无查看明文入口 |
| 弹窗关闭后表单重置 | 所有错误提示清除，输入值恢复默认 |
| 弹窗遮罩可关闭 | 点击半透明遮罩区域等同于点击取消 |
| 验证码演示模式 | 原型阶段不实际发送短信，输入任意 6 位数字即可通过 |
| 登录记录仅展示不翻页 | 固定展示最近 10 条记录，无分页控件 |
