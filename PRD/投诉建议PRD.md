## 一、模块概述

### 1.1 模块目标

集中管理用户投诉与建议反馈：接收用户端提交的反馈单，运营受理、回复、关闭，形成反馈闭环，提升用户信任与服务质量。

### 1.2 与售后列表的边界

| 维度 | 售后列表 | 投诉建议 |
|------|---------|---------|
| 关联对象 | 订单 Order | 用户 User（非订单维度） |
| 来源 | 用户售后/运营标记/系统生成 | 用户端投诉建议页提交 |
| 核心操作 | 处理结果登记（退款/赔付/开柜） | 运营回复 + 状态流转 |
| 金额记录 | 有（退款金额/赔付金额） | 无 |
| 数据对象 | Ticket（售后/异常单） | Feedback（反馈单） |

### 1.3 页面构成

| 页面 | 说明 |
|------|------|
| 投诉建议列表 | 筛选 + KPI 指标卡 + 表格 + 分页 |
| 反馈详情/处理抽屉 | 右侧滑出，查看反馈详情 + 运营回复 + 状态操作 |

### 1.4 入口

- 一级菜单「用户管理」→ 二级「投诉建议」；
- 工作台待办（待处理投诉建议数）可带筛选跳入。

## 二、反馈单状态流转（4 态）

```
提交反馈 → 待处理 → 处理中 → 已回复 → 已关闭
```

| 状态 | 含义 | Badge 色 | 运营可执行操作 |
|------|------|----------|---------------|
| 待处理 | 用户已提交，等待运营受理 | warning `#E7772D` | 受理（→处理中） |
| 处理中 | 运营已受理，正在核实处理 | info `#0091D5` | 提交回复（→已回复）、关闭（→已关闭） |
| 已回复 | 运营已给出回复内容 | success `#0FC060` | 追加回复（保持已回复）、关闭（→已关闭） |
| 已关闭 | 反馈关闭的终态 | ink-muted 灰 `#9AA4A1` | 无（只读） |

**关闭规则**「常见逻辑补充」：① 已回复后 7 天无追问自动关闭；② 反馈信息不足且无法联系用户时运营手动关闭（附关闭原因）。已关闭记录保留可查。

## 四、页面二：反馈详情/处理抽屉

### 4.1 目标

查看完整反馈信息、用户信息，运营回复处理，状态流转。

### 4.2 抽屉结构（右侧滑出，800px）

| 区块 | 内容 |
|------|------|
| 标题栏 | "反馈详情" + 反馈编号 + 关闭 × |
| 状态卡 | 类型 Badge + 分类 + 状态 Badge + 状态说明文案 |
| 反馈内容卡 | 完整反馈内容、图片凭证（可点击放大查看）、联系方式（有则展示） |
| 用户信息卡 | 用户手机号（脱敏）、注册时间、（可点击跳转用户列表查看该用户） |
| 运营回复区 | 见 4.3 |
| 处理记录 | 状态变更时间线（提交→受理→回复→关闭） |
| 状态操作按钮 | 见 4.4 |

### 4.3 运营回复区

| 场景 | 展示内容 |
|------|----------|
| 待处理 / 处理中（无回复） | 回复表单：textarea（上限 500 字，带字数计数器）+ 「提交回复并完成」按钮 |
| 已回复 / 已关闭（有回复） | 绿底气泡展示回复内容 + 回复时间 + 回复人；可「追加回复」（展开表单） |

### 4.4 状态操作按钮

| 当前状态 | 可见按钮 |
|----------|----------|
| 待处理 | `[受理]` → 待处理→处理中 |
| 处理中 | `[提交回复]` → 处理中→已回复；`[关闭]` → 弹出关闭原因弹窗→已关闭 |
| 已回复 | `[追加回复]` → 保持已回复；`[关闭]` → 弹出关闭原因弹窗→已关闭 |
| 已关闭 | 无操作按钮（只读），显示关闭原因 |

### 4.5 关闭原因弹窗

关闭时弹出 Dialog（confirm 类型），必填关闭原因（下拉选择：信息不足无法联系 / 已解决 / 重复反馈 / 其他），提交后状态变更为已关闭。

## 五、业务规则

- 反馈单独立于订单，一个用户可有多条反馈单；
- 反馈单号规则：`FB` + 日期（YYYYMMDD）+ 3 位序号，如 `FB20240714001`；
- 状态 4 态：待处理 → 处理中 → 已回复 → 已关闭；
- 运营回复后用户端可见回复内容（通过服务号通知用户）；
- 已回复 7 天无追问自动关闭；已关闭不可重开，用户可重新提交新反馈；
- 涉及具体订单退款/赔付诉求时，运营回复中引导用户走售后通道；
- 处理记录留痕，不可删除；
- 权限：管理员与运营员均可查看与处理。
