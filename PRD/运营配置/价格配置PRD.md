# 价格配置 PRD（后台 / 运营配置）

**文档版本**：v1.0
**页面名称**：价格配置（Pricing Configuration）
**所属模块**：后台 / 运营配置 / 价格配置
**适用端**：PC Web 后台
**文档状态**：从 运营配置PRD.md 拆分
**创建日期**：2026-07-09

---

## 1. 页面概述

价格配置是运营配置模块的子功能，供**管理员**配置全站计费规则。**一期为全局统一一套价**（不按柜机/网点差异化，差异化留 V2）。此配置是用户端下单页、收费规则弹窗、订单退款计算、后台对账的**唯一计费依据**，不得在各端硬编码。

> **权限**：仅管理员可操作。运营员不可见本模块。

---

## 2. 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"价格配置" + "生效范围：全局"标识 |
| B 短租价格 | 按柜型三档：单价 + 每24h封顶 |
| C 长租价格 | 按柜型三档：月租 |
| D 超时费率 | 短租超时规则说明 + 长租按柜型日费率 |
| E 退款规则参数 | 短租/长租退款阈值与比例阶梯 |
| F 保存操作 | 保存（二次确认）+ 变更记录入口 |

---

## 3. 区块 B：短租价格

| 柜型 | 单价（元/小时） | 每24小时封顶（元） |
|------|----------------|--------------------|
| 小格口 | 2 | 9 |
| 中格口 | 2.5 | 12 |
| 大格口 | 3.5 | 17 |

- 每档为可编辑数字输入（mono），带单位；
- 输入校验：正数、合理范围、封顶 ≥ 单价。

---

## 4. 区块 C：长租价格

| 柜型 | 月租（元/月，每月按31天计） |
|------|------------------------------|
| 小格口 | 25 |
| 中格口 | 30 |
| 大格口 | 40 |

- 可编辑数字输入；
- 页面注明"每月按 31 天计"。

---

## 5. 区块 D：超时费率

| 参数 | 说明 | 可配 |
|------|------|------|
| 短租超时计费方式 | 按当前格口短租单价计算 | 可配：计费单位（按小时）、不足单位进位规则（不足1小时按1小时） |
| 计费上限 | 上不封顶 | 可配开关（是否封顶 + 封顶值） |

---

## 6. 区块 E：退款规则参数

**短租退款算法**

| 参数 | 说明 | 可配 |
|------|------|------|
| 全额退时限 | 下单后 N 分钟内全额退 | 可配（分钟，默认5） |
| 退款算法 | 退款=实付−使用时长费用 | 可配：使用时长费用的计算基准（按短租单价）、费用≥实付时是否不退（可配开关，默认不退） |

**长租退款**

| 参数 | 说明 | 可配 |
|------|------|------|
| 全额退时限 | 下单后 N 分钟内 | 可配（默认5分钟） |
| 整月退费 | 剩余整月按月租整月退 | 可配开关 |
| 不足整月比例阶梯 | ≤1天50% / 1-2天30% / 2-3天10% / 3-31天不退 | **各时段区间与各比例均可配**（可增删阶梯行） |

> 退款比例阶梯做成**可增删行的表格**（每行=时段区间+退款比例%），便于运营调整档位，不写死四档。

---

## 7. 区块 F：保存与变更留痕

| 操作 | 响应 |
|------|------|
| 点击"保存" | **二次确认弹窗**："价格修改将立即全局生效，影响用户端下单、退款与对账，确认保存？" |
| 确认保存 | 保存并全局生效 + **记录变更留痕**（操作人、时间、变更前后对比） |
| 取消 | 不保存 |
| 查看变更记录 | 打开变更历史（谁在何时把某项从 X 改为 Y） |

**变更留痕字段**：变更时间、操作人、变更项、原值、新值。留痕不可删除，供审计与纠纷追溯。

保存价格时，增加**生效时间选择**：

| 选项 | 说明 |
|------|------|
| 立即生效 | 保存后即时全局生效 |
| 定时生效 | 指定未来某时间点生效；生效前用户端仍用旧价 |

保存流程：选择生效方式 → 二次确认（含生效时间提示）→ 保存 → 变更留痕（记录生效时间）。定时生效的配置在生效前可展示"待生效"状态并可取消/修改。

**变更留痕字段增补**：变更时间、操作人、变更项、原值、新值、**生效方式（立即/定时）、生效时间**。

---

## 8. 业务规则

- **计费单一事实来源**：本配置是唯一计费依据，用户端下单页、收费规则弹窗、订单退款、对账均引用，禁止硬编码；
- 一期**全局一套价**，不按柜机差异化（V2）；
- 价格需与小铁柜机实际计费对齐，避免实付与展示不一致；
- 保存即全局生效，需二次确认 + 变更留痕；
- 仅管理员可操作。
