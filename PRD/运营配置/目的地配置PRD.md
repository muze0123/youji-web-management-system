# 目的地配置 PRD（后台 / 运营配置）

**文档版本**：v1.0
**页面名称**：目的地配置（Destination Configuration）
**所属模块**：后台 / 运营配置 / 目的地配置
**适用端**：PC Web 后台
**文档状态**：从 运营配置PRD.md 拆分
**创建日期**：2026-07-09

---

## 1. 页面概述

目的地配置是运营配置模块的子功能，供**管理员**维护行李寄送的固定目的地清单，供用户端寄送时选择。

> **权限**：仅管理员可操作。运营员不可见本模块。

---

## 2. 页面结构

| 区块 | 说明 |
|------|------|
| A 页面头 | 面包屑 + 标题"目的地配置" + "新增目的地"按钮 |
| B 筛选/搜索 | 按类型/启用状态筛选、按名称搜索 |
| C 目的地列表表格 | 主体 |
| D 新增/编辑弹窗 | 增改目的地 |

---

## 3. 列表字段

| 列 | 说明 |
|----|------|
| 目的地名称 | 如"XX 酒店""XX 火车站" |
| 地址 | 详细地址 |
| 类型 | 酒店/车站/景区/其他 |
| 服务费 | 一口价（元，mono） |
| 预计时效 | 预计送达时长 |
| 启用状态 | 启用/停用（Badge） |
| 操作 | 编辑、启用/停用、删除 |

---

## 4. 新增/编辑弹窗字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 目的地名称 | 必填 | — |
| 地址 | 必填 | — |
| 类型 | 必填 | 酒店/车站/景区/其他 |
| 服务费 | 必填 | 一口价（元） |
| 预计时效 | 必填 | 如"约4小时内送达" |
| 启用状态 | — | 默认启用 |

---

## 5. 交互

| 操作 | 响应 |
|------|------|
| 新增目的地 | 打开弹窗，填写保存 |
| 编辑 | 打开弹窗回填修改 |
| 启用/停用 | 切换状态；**停用后用户端寄送不展示该目的地** |
| 删除 | 二次确认后**软删除**——从用户端与可选清单移除，但后台保留记录，历史订单的目的地关联不丢失 |
| 筛选/搜索 | 列表过滤 |

> 软删除的目的地不出现在用户端寄送选择与后台正常列表，但可通过"已删除"筛选查看，保证历史订单可追溯。

---

## 6. 业务规则

- 目的地为用户端行李寄送目的地选择的来源；
- 停用即时对用户端隐藏；
- 服务费为该目的地一口价（与寄送计费一致）；
- 仅管理员可操作。

---

## 7. 原型交互标注要点

| 标注点 | 说明要点 |
|--------|----------|
| 目的地启用/停用 | 停用即时对用户端隐藏 |
| 目的地 CRUD | 新增/编辑/删除目的地的交互规则 |
| 目的地删除 | 软删除，历史订单关联不丢失 |
| 整模块权限 | 仅管理员可见可操作 |
