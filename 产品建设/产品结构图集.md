# 第二部分　产品信息结构图

> **定义**：脱离实际页面，将产品的数据对象抽象出来并列出其字段，是产品的"数据抽象"。秉承"万物皆对象"的理念。
> **用途**：给开发建表参考。同一对象的信息可跨多个页面复用，故按数据对象组织，而非按页面罗列。

产品的核心数据对象共 8 个：订单、柜点、格口、用户、取件码、异常工单、支付记录、对账记录。

## 对象一：订单 Order（核心对象）

| 字段 | 说明 |
|------|------|
| 订单号 | 唯一标识（主键） |
| 业务类型 | 寄存 / 寄送 |
| 订单状态 | 待支付/已支付/寄存中/待转运/已完成/已取消/异常等 |
| 订单来源 | 小铁小程序 / 自有小程序 |
| 用户 ID | 关联下单用户 |
| 机柜编码 | 关联柜点 |
| 格口名称 | 关联具体格口 |
| 柜型 | 小/中/大/超大 |
| 支付金额 | 单位：分 |
| 下单时间 | 创建时间 |
| 结束时间 | 预期/真正结束时间 |
| 使用时长 | 单位：分钟 |
| 是否超时 | 布尔 |
| 取件码 | 关联取件凭证 |
| 目的地 | 寄送业务专属 |
| 收件信息 | 寄送业务专属 |
| 预计送达 | 寄送业务专属 |

## 对象二：柜点 Cabinet（资源对象）

| 字段 | 说明 |
|------|------|
| 机柜编码 | 唯一标识（主键） |
| 机柜名称 | 展示名 |
| 位置信息 | 如"广州体育馆西广场(A)" |
| 运营状态 | 可用 / 停用 |
| 在线状态 | 在线 / 离线 |
| 网点名称 | 所属网点 |
| 省 / 市 / 区 | 行政区划 |
| 详细地址 | 完整地址 |
| 经纬度 | 用于定位与距离排序 |
| 上下线时间 | 状态记录 |

## 对象三：格口 Locker（资源对象）

| 字段 | 说明 |
|------|------|
| 格口编号 | 如"006" |
| 所属机柜 | 关联机柜编码 |
| 柜型 | 小/中/大/超大/其他 |
| 可用状态 | 空闲/占用/维护中 |
| 运营状态 | 启用 / 停用 |
| 计费规则 | 本地维护的柜型价格映射 |

## 对象四：用户 User（主体对象）

| 字段 | 说明 |
|------|------|
| 用户 ID | 唯一标识（主键） |
| 微信 OpenID | 微信授权标识 |
| 昵称 | 用户昵称 |
| 手机号 | 脱敏存储 |
| 注册时间 | 首次授权时间 |
| 历史订单 | 关联订单列表 |
| 会员等级 | V2 引入 |

## 对象五：取件码 PickupCode（凭证对象）

| 字段 | 说明 |
|------|------|
| 取件码 | 取件凭证（加密存储） |
| 关联订单 | 所属订单 |
| 关联格口 | 所绑定格口 |
| 有效期 | 取件时限 |
| 核销状态 | 未核销 / 已核销 |
| 二维码 | 配合数字码的扫码凭证 |

## 对象六：异常工单 Ticket（业务对象）

| 字段 | 说明 |
|------|------|
| 工单号 | 唯一标识（主键） |
| 关联订单 | 所属订单 |
| 异常类型 | 延误/损坏/遗失/取件码错误/支付异常等 |
| 工单状态 | 待处理/处理中/已处理/已关闭 |
| 处理人 | 运营人员 |
| 处理备注 | 处理记录留痕 |
| 创建/更新时间 | 时间记录 |

## 对象七：支付记录 Payment（业务对象）

| 字段 | 说明 |
|------|------|
| 支付单号 | 唯一标识（主键） |
| 关联订单 | 所属订单 |
| 支付金额 | 实付金额 |
| 支付方式 | 微信 / 支付宝 |
| 支付状态 | 待支付/成功/失败/已退款 |
| 收款方 | 小铁（一期）/ 公司（二期） |
| 退款状态 | 无 / 退款中 / 已退款 |
| 支付时间 | 时间记录 |

## 对象八：对账记录 Reconcile（业务对象）

| 字段 | 说明 |
|------|------|
| 对账批次 | 唯一标识（主键） |
| 对账日期 | 对账周期 |
| 订单数 | 当期订单总数 |
| 本地金额 | 本系统记录金额 |
| 小铁金额 | 小铁结算金额 |
| 差异金额 | 两者差额 |
| 差异状态 | 一致 / 待核查 / 已确认 |

# 第三部分　产品结构图

> **定义**：= 功能 + 信息 + 基本交互逻辑，是原型的低成本替代。说清每个页面的功能、展示字段与页面跳转逻辑。
> **用途**：需求评审时替代原型，方便快速增删改，避免过早投入原型制作。

每个页面包含三要素：**主要功能**、**展示字段**、**跳转去向**。
